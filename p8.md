## 1. Purpose

Build the orchestration layer that drives the adapter → storage pipeline on schedule. P8 owns:

1. **DAG generation** — cadence-grouped Airflow DAGs with per-tier task groups, generated from `stream_registry.yaml`.
2. **Task execution** — the `TaskRunner` that wires `BaseAdapter.run()` → `StorageRouter.route()` per stream, with auth, error handling, and metrics.
3. **Concurrency control** — global and per-cadence adapter execution limits.
4. **Backpressure** — WAQ depth monitoring with soft (delay) and hard (reject) thresholds per engine.
5. **Health integration** — scheduler-level health aggregating `AdapterHealth`, `StorageHealth`, and queue state.
6. **Failure handling** — per-stream retry, SLA alerting, and dead-stream detection.

After P8, the platform runs autonomously: streams are fetched on cadence, records flow through the pipeline, and failures are surfaced.

### 3.3 Execution Flow

```
Airflow Scheduler
    │
    ▼
DAG trigger (cadence-based)
    │
    ▼
TaskGroup: tier_N
    │
    ├── Pre-check: BackpressureMonitor.check()
    │       ├── all engines below soft → proceed
    │       ├── any engine at soft → delay (configurable wait, re-check)
    │       └── any engine at hard → skip task, log WARNING, mark SKIPPED
    │
    ├── Pre-check: ConcurrencyManager.acquire()
    │       ├── slot available → proceed
    │       └── no slot → wait (up to timeout), then fail
    │
    ▼
TaskRunner.execute(stream_id)
    │
    ├── 1. Instantiate adapter (registry lookup → adapter class)
    ├── 2. Inject AuthManager credentials
    ├── 3. adapter.run() → list[NormalizedRecord]
    ├── 4. StorageRouter.route(records) → RouteResult
    ├── 5. Log metrics (records fetched, routed, deduplicated, duration)
    ├── 6. ConcurrencyManager.release()
    └── 7. Return result to Airflow (success/failure/skip)
```

## 5. Task Runner

Location: `src/hydra/scheduler/task_runner.py`

### 5.1 Class Definition

```python
class TaskRunner:
    """
    Executes a single adapter stream: fetch → parse → validate → normalize → route.

    Wires together BaseAdapter, AuthManager, and StorageRouter.
    Called by Airflow PythonOperator tasks.
    """

    def __init__(
        self,
        registry: StreamRegistry,
        auth_manager: AuthManager,
        storage_router: StorageRouter,
        backpressure_monitor: BackpressureMonitor,
        concurrency_manager: ConcurrencyManager,
        settings: HydraSettings,
    ):
        self._registry = registry
        self._auth = auth_manager
        self._router = storage_router
        self._backpressure = backpressure_monitor
        self._concurrency = concurrency_manager
        self._settings = settings
        self._adapter_map: dict[str, type[BaseAdapter]] = {}  # populated at init

    async def execute(self, stream_id: str, **context) -> TaskResult:
        """
        Full pipeline execution for a single stream.

        Steps:
        1. Backpressure pre-check
        2. Concurrency slot acquisition
        3. Adapter instantiation (from registry adapter_type)
        4. Auth injection
        5. adapter.run()
        6. StorageRouter.route(records)
        7. Metrics emission
        8. Concurrency slot release

        Returns TaskResult with execution metadata.
        Raises TaskSkipped if backpressure rejects.
        Raises TaskFailed on unrecoverable adapter/storage errors.
        """
        ...

    def _resolve_adapter_class(self, adapter_type: str) -> type[BaseAdapter]:
        """
        Map adapter_type string from registry to concrete adapter class.

        Mapping:
        - rest_json → RestJsonAdapter
        - fdsn → FdsnAdapter
        - ckan → CkanAdapter
        - odata → ODataAdapter
        - sdmx → SdmxAdapter
        - tap_vo → TapVoAdapter
        - s3_bulk → S3BulkAdapter
        - scrape_rss → ScrapeRssAdapter
        - ais_adsb → AisAdsbAdapter
        - stix_taxii → StixTaxiiAdapter
        """
        ...

    async def _execute_with_fallback(
        self, stream_id: str, tier: StreamTier
    ) -> list[NormalizedRecord]:
        """
        Try primary adapter. On FetchError, try fallback adapter if declared.

        Fallback is attempted once — no recursive fallback chains.
        Both attempts logged with adapter_type and outcome.
        """
        ...
```

### 5.2 TaskResult Dataclass

```python
@dataclass
class TaskResult:
    stream_id: str
    adapter_type: str
    status: Literal["success", "partial", "skipped", "failed"]
    records_fetched: int
    records_routed: int
    records_deduplicated: int
    records_failed: int
    route_result: RouteResult | None
    duration_ms: float
    error: str | None = None
    fallback_used: bool = False
    backpressure_delayed: bool = False
    timestamp: str = ""  # ISO 8601 UTC
```

### 5.3 Adapter Instantiation

The `TaskRunner` maintains a mapping of `adapter_type` strings (from `stream_registry.yaml`) to concrete adapter classes. On `execute()`:

1. Look up `stream_id` in registry → get `StreamTier` → get `adapter` type string.
2. Resolve to concrete class via `_resolve_adapter_class()`.
3. Instantiate: `adapter = AdapterClass(stream_id=stream_id, settings=self._settings)`.
4. Apply auth: `auth_request = await self._auth.apply(stream_id)` — adapter receives authenticated request context.
5. Execute: `records = await adapter.run()`.

### 5.4 Fallback Execution

If the primary adapter raises `FetchError` (not `ParseError`, `ValidationError`, or `RateLimitError`), and the tier declares a `fallback` adapter type:

1. Log WARNING: `{stream_id, primary_adapter, error, action: "fallback_attempt"}`.
2. Instantiate fallback adapter.
3. Apply auth (same `stream_id`, potentially different auth pattern).
4. Execute fallback adapter.
5. If fallback succeeds: `TaskResult.fallback_used = True`, proceed to routing.
6. If fallback fails: `TaskResult.status = "failed"`, both errors logged.

`RateLimitError` does **not** trigger fallback — the upstream is responsive but throttling. Retrying with a different adapter type won't help and may violate rate limits on the fallback source.

### 5.5 Airflow Integration

Each stream task is a `PythonOperator` (or `AsyncPythonOperator` if available in the Airflow version) that calls:

```python
def run_stream_task(stream_id: str, **context):
    """Airflow PythonOperator callable."""
    runner = _get_task_runner()  # singleton per worker process
    result = asyncio.run(runner.execute(stream_id, **context))

    if result.status == "skipped":
        raise AirflowSkipException(f"Backpressure skip: {result.error}")
    elif result.status == "failed":
        raise AirflowException(f"Stream failed: {result.error}")

    # Push metrics to XCom for downstream visibility
    context["ti"].xcom_push(key="task_result", value=asdict(result))
```

The `_get_task_runner()` factory initializes shared resources (registry, auth manager, storage router, backpressure monitor, concurrency manager) once per Airflow worker process and caches them.

## 6. Concurrency Control

Location: `src/hydra/scheduler/concurrency.py`

### 6.1 Design

Per design decision #3: **global cap of 10 with per-cadence sub-limits configurable**.

```python
class ConcurrencyManager:
    """
    Semaphore-based adapter execution limiter.

    Two-level limiting:
    1. Global semaphore — caps total concurrent adapter.run() calls across all DAGs.
    2. Per-cadence semaphore — caps concurrent runs within a single cadence tier.

    Both levels must be acquired before execution proceeds.
    """

    def __init__(self, settings: HydraSettings):
        self._global_limit = settings.scheduler.global_concurrency_limit  # default: 10
        self._cadence_limits = settings.scheduler.cadence_concurrency_limits
        self._global_sem: asyncio.Semaphore = asyncio.Semaphore(self._global_limit)
        self._cadence_sems: dict[str, asyncio.Semaphore] = {}
        self._active_count: int = 0
        self._active_by_cadence: dict[str, int] = {}

    async def acquire(self, cadence: str, timeout: float = 60.0) -> bool:
        """
        Acquire both global and cadence semaphore slots.

        Returns True if acquired within timeout, False otherwise.
        On timeout: log WARNING, return False (caller should fail the task).
        """
        ...

    async def release(self, cadence: str) -> None:
        """Release both semaphore slots."""
        ...

    @property
    def active_count(self) -> int:
        """Current number of active adapter executions."""
        return self._active_count

    def active_by_cadence(self, cadence: str) -> int:
        """Current active count for a specific cadence."""
        return self._active_by_cadence.get(cadence, 0)
```

### 6.2 Default Cadence Concurrency Limits

| Cadence | Default Limit | Rationale |
|---|---|---|
| `sub_minute` | 4 | High-frequency, must leave headroom for other cadences |
| `realtime` | 3 | Same rationale as sub_minute |
| `15min` | 3 | Moderate frequency |
| `hourly` | 4 | Larger batch, more streams |
| `daily` | 6 | Largest tier count, runs once/day — can use more slots |
| `weekly` | 4 | Moderate tier count |
| `monthly_plus` | 2 | Low frequency, low priority |

Note: per-cadence limits are soft caps within the global cap of 10. If `sub_minute` has 4 slots and `daily` has 6, they cannot both saturate simultaneously — the global cap of 10 constrains the sum.

### 6.3 Configuration

New fields in `HydraSettings.scheduler`:

```python
global_concurrency_limit: int = 10
cadence_concurrency_limits: dict[str, int] = {
    "sub_minute": 4,
    "realtime": 3,
    "15min": 3,
    "hourly": 4,
    "daily": 6,
    "weekly": 4,
    "monthly_plus": 2,
}
```

Overridable via `stream_registry.yaml` per-tier `max_concurrent` field (optional, defaults to cadence limit).

### 6.4 Airflow Pool Integration

In addition to the application-level semaphores, Airflow Pools provide a secondary enforcement layer:

- Pool `hydra_global`: 10 slots — mirrors the global concurrency limit.
- Pool `hydra_cadence_{cadence}`: per-cadence slot count — mirrors cadence limits.

Each `PythonOperator` task is assigned to both pools. This provides Airflow-native visibility into concurrency usage via the Airflow UI and prevents over-scheduling even if multiple worker processes are active.

## 7. Backpressure

Location: `src/hydra/scheduler/backpressure.py`

### 7.1 Design

Per design decision #4: **per-engine thresholds — soft 1,000 (delay), hard 5,000 (reject), configurable in HydraSettings**.

```python
class BackpressureMonitor:
    """
    Monitors Redis WAQ depths and makes throttle/reject decisions.

    Three states per engine:
    - CLEAR: queue depth < soft_limit → proceed normally
    - THROTTLED: soft_limit <= queue depth < hard_limit → delay execution
    - BLOCKED: queue depth >= hard_limit → reject execution
    """

    def __init__(self, redis_cache: RedisCache, settings: HydraSettings):
        self._redis = redis_cache
        self._settings = settings
        self._engine_queues = {
            "postgres": "hydra:waq:postgres",
            "influxdb": "hydra:waq:influxdb",
            "elasticsearch": "hydra:waq:elasticsearch",
            "neo4j": "hydra:waq:neo4j",
            "minio": "hydra:waq:minio",
        }

    async def check(self) -> BackpressureState:
        """
        Check all engine queue depths.

        Returns aggregate state:
        - CLEAR if all engines below soft limit
        - THROTTLED if any engine between soft and hard
        - BLOCKED if any engine at or above hard limit
        """
        ...

    async def check_engine(self, engine: str) -> EngineBackpressure:
        """Check a single engine's queue depth against thresholds."""
        ...

    async def wait_for_clear(
        self, max_wait: float = 60.0, poll_interval: float = 5.0
    ) -> bool:
        """
        Block until backpressure clears or timeout.

        Used when state is THROTTLED — wait for queues to drain.
        Returns True if cleared, False if timed out.
        """
        ...
```

### 7.2 Backpressure Dataclasses

```python
@dataclass
class EngineBackpressure:
    engine: str
    queue_depth: int
    soft_limit: int
    hard_limit: int
    state: Literal["CLEAR", "THROTTLED", "BLOCKED"]

@dataclass
class BackpressureState:
    overall: Literal["CLEAR", "THROTTLED", "BLOCKED"]
    engines: dict[str, EngineBackpressure]
    checked_at: str  # ISO 8601 UTC
```

### 7.3 Threshold Configuration

New fields in `HydraSettings.scheduler`:

```python
backpressure_soft_limit: int = 1_000    # per-engine queue depth triggering delay
backpressure_hard_limit: int = 5_000    # per-engine queue depth triggering reject
backpressure_wait_timeout: float = 60.0  # max seconds to wait when throttled
backpressure_poll_interval: float = 5.0  # seconds between re-checks when waiting
```

Per-engine overrides (optional, in `HydraSettings.scheduler.engine_backpressure_overrides`):

```python
engine_backpressure_overrides: dict[str, dict[str, int]] = {}
# Example: {"postgres": {"soft_limit": 2000, "hard_limit": 10000}}
```

### 7.4 Integration with TaskRunner

In `TaskRunner.execute()`:

```python
# Step 1: Backpressure pre-check
bp_state = await self._backpressure.check()

if bp_state.overall == "BLOCKED":
    return TaskResult(
        stream_id=stream_id,
        status="skipped",
        error=f"Backpressure BLOCKED: {self._format_bp_detail(bp_state)}",
        ...
    )

if bp_state.overall == "THROTTLED":
    cleared = await self._backpressure.wait_for_clear()
    if not cleared:
        return TaskResult(
            stream_id=stream_id,
            status="skipped",
            error=f"Backpressure THROTTLED timeout: {self._format_bp_detail(bp_state)}",
            backpressure_delayed=True,
            ...
        )
    # Cleared — proceed with execution
    result.backpressure_delayed = True
```

### 8.3 Dead Stream Detection

A stream is classified as "dead" when it has failed `N` consecutive scheduled runs (default: 5, configurable via `HydraSettings.scheduler.dead_stream_threshold`).

Tracking mechanism: Redis hash `hydra:stream_failures:{stream_id}` with fields:
- `consecutive_failures`: int
- `last_failure_at`: ISO 8601
- `last_error`: string

On successful `TaskRunner.execute()`: reset counter to 0.
On failed execution: increment counter.
When counter reaches threshold: add to `dead_streams` list in health report, log at ERROR.

Dead streams are **not** automatically disabled — they continue to be scheduled (the upstream may recover). The health report surfaces them for operator attention. P12 wires this to alerting.

## 9. Maintenance DAG

Location: `dags/maintenance.py`

A daily DAG that runs operational health tasks:

### 9.1 Tasks

**`health_check_all_adapters`**
- Iterates all streams in registry.
- Calls `adapter.health_check()` for each.
- Stores results in Redis: `hydra:adapter_health:{stream_id}`.
- Logs DEGRADED/UNREACHABLE adapters at WARNING.

**`health_check_all_storage`**
- Calls `StorageHealthAggregator.check_all()`.
- Logs per-engine status.
- If any primary engine (PG, Redis) is UNREACHABLE: log CRITICAL.

**`backpressure_report`**
- Calls `BackpressureMonitor.check()`.
- Logs per-engine queue depths.
- If any engine THROTTLED or BLOCKED: log WARNING/ERROR.

**`dead_stream_detection`**
- Scans all `hydra:stream_failures:*` keys.
- Identifies streams exceeding `dead_stream_threshold`.
- Logs dead streams at ERROR with last error detail.

**`dlq_depth_alert`**
- Checks DLQ depth for all engines via `RedisCache.dlq_depth()`.
- If any DLQ exceeds `reconciliation_alert_threshold` (from P7 settings): log ERROR with engine name, depth, and threshold.
- Pushes DLQ depth metrics to XCom for P12 alerting integration.

## 10. Failure Handling

### 10.1 Retry Strategy

Airflow-native retries handle transient failures:

- **Default retries:** 3 (from `DEFAULT_DAG_ARGS`)
- **Retry delay:** 2 minutes initial, exponential backoff, max 30 minutes
- **Execution timeout:** per-cadence (§4.3)

Adapter-level retries (P1 exponential backoff: 1s initial, 2x factor, max 5 retries, 60s ceiling) execute **within** a single Airflow task attempt. Airflow retries are the outer loop — they re-execute the entire `TaskRunner.execute()` pipeline.

### 10.2 Exception → Airflow State Mapping

| Exception | Airflow State | Retry? | Notes |
|---|---|---|---|
| `FetchError` (after fallback exhausted) | `FAILED` | Yes (Airflow retry) | Transient network/upstream failure |
| `RateLimitError` | `UP_FOR_RETRY` | Yes (with extended delay) | Adapter-level backoff already attempted |
| `ParseError` | `FAILED` | Yes (1 retry only) | Unlikely to self-resolve; may indicate schema drift |
| `ValidationError` | `FAILED` | No (`max_retries=0` override) | Data quality issue — retrying won't help |
| `BackpressureBlocked` | `SKIPPED` | No | WAQ saturated — next scheduled run will retry |
| `ConcurrencyTimeout` | `FAILED` | Yes | Slot contention — may clear on retry |
| `AuthenticationError` (from P6) | `FAILED` | Yes (1 retry only) | Token may have expired; 1 retry allows refresh |
| Unhandled exception | `FAILED` | Yes (Airflow retry) | Defensive — log full traceback |

### 10.3 RateLimitError Special Handling

When `RateLimitError` is caught:

1. Extract `retry_after` from the error (set by P1 adapter from HTTP `Retry-After` header).
2. If `retry_after` is available and ≤ execution timeout: `await asyncio.sleep(retry_after)`, then re-execute adapter.
3. If `retry_after` exceeds execution timeout or is unavailable: raise to Airflow for retry with extended delay (`retry_delay * 3`).

## 12. Exceptions

Location: `src/hydra/scheduler/exceptions.py`

```python
class SchedulerError(Exception):
    """Base exception for scheduler module."""

class BackpressureBlocked(SchedulerError):
    """Raised when WAQ depth exceeds hard limit — task should be skipped."""
    def __init__(self, engine: str, depth: int, hard_limit: int):
        self.engine = engine
        self.depth = depth
        self.hard_limit = hard_limit
        super().__init__(f"Backpressure BLOCKED: {engine} depth={depth} >= hard_limit={hard_limit}")

class ConcurrencyTimeout(SchedulerError):
    """Raised when concurrency slot acquisition times out."""
    def __init__(self, cadence: str, timeout: float):
        self.cadence = cadence
        self.timeout = timeout
        super().__init__(f"Concurrency timeout: {cadence} after {timeout}s")

class AdapterResolutionError(SchedulerError):
    """Raised when adapter_type string cannot be mapped to a concrete class."""
    def __init__(self, adapter_type: str):
        self.adapter_type = adapter_type
        super().__init__(f"Unknown adapter type: {adapter_type}")

class DeadStreamError(SchedulerError):
    """Raised when a stream exceeds the consecutive failure threshold."""
    def __init__(self, stream_id: str, consecutive_failures: int, threshold: int):
        self.stream_id = stream_id
        self.consecutive_failures = consecutive_failures
        self.threshold = threshold
        super().__init__(
            f"Dead stream: {stream_id} failed {consecutive_failures} consecutive runs (threshold: {threshold})"
        )
```
