
# P1 — Base Adapter Interface & REST/JSON Adapter

## Kiro Specification — HYDRA OSINT Platform
**Version:** 0.1.0
**Phase:** 1 (MVP)
**Depends on:** P0 (Project Scaffold & Shared Contracts)

## 1. Purpose

Define the abstract base adapter interface that all HYDRA ingestion adapters implement, and deliver the first concrete adapter — Generic REST/JSON — covering approximately 60% (~84 streams) of the platform's data sources. Every subsequent adapter (P2–P5) inherits from this base.

---

## 2. Integration Contracts (from P0)

- **NormalizedRecord** — `src/hydra/models/normalized.py` — the universal output type for all adapters.
- **stream_registry.yaml** — `src/hydra/registry/stream_registry.yaml` — authoritative stream metadata (all 28 tiers). Each stream entry contains: `stream_id`, `tier`, `adapter_type`, `base_url`, `auth_pattern`, `poll_cadence`, `format`, `parser`, and optional fields.
- **HydraSettings** — `from hydra.config import settings` — centralized configuration including credential references, timeouts, retry policy.
- **compute_raw_hash** — `from hydra.utils.hashing import compute_raw_hash` — deterministic hash for deduplication of raw payloads.

## 3. Base Adapter Interface — `src/hydra/adapters/base.py`

### 3.1 Class: `BaseAdapter` (abstract)

An abstract base class. All adapters subclass it. No adapter may bypass this interface.

**Constructor parameters:**
- `stream_id: str` — must match a key in `stream_registry.yaml`.
- `settings: HydraSettings` — injected configuration.
- On init, the adapter loads its own stream metadata from the registry and validates that the declared `adapter_type` matches the subclass type. Raise `AdapterRegistryMismatch` if not.

**Abstract methods (must be implemented by every subclass):**

| Method | Signature | Returns | Description |
|---|---|---|---|
| `fetch` | `async def fetch(self) -> RawPayload` | `RawPayload` | Retrieve raw data from the upstream source. Must handle HTTP errors, timeouts, and rate limits internally. Returns a `RawPayload` dataclass. |
| `parse` | `def parse(self, raw: RawPayload) -> list[dict]` | `list[dict]` | Transform raw bytes/text into a list of Python dicts — one dict per discrete record. No normalization yet. |
| `validate` | `def validate(self, records: list[dict]) -> list[dict]` | `list[dict]` | Apply stream-specific validation rules. Drop or flag malformed records. Return only valid records. Log dropped records at WARNING level. |

**Concrete methods (implemented in base, inherited by all):**

| Method | Signature | Returns | Description |
|---|---|---|---|
| `normalize` | `def normalize(self, records: list[dict]) -> list[NormalizedRecord]` | `list[NormalizedRecord]` | Map validated dicts to `NormalizedRecord` instances using a field-mapping declared in the stream registry entry. Calls `compute_raw_hash` on each record's raw representation for deduplication. |
| `run` | `async def run(self) -> list[NormalizedRecord]` | `list[NormalizedRecord]` | Orchestrator: calls `fetch → parse → validate → normalize` in sequence. Wraps the full pipeline in structured logging (start, record counts at each stage, duration, errors). Returns the final normalized list. |
| `health_check` | `async def health_check(self) -> AdapterHealth` | `AdapterHealth` | Lightweight probe of the upstream endpoint (HEAD request or equivalent). Returns an `AdapterHealth` dataclass with `status` (enum: `OK`, `DEGRADED`, `UNREACHABLE`), `latency_ms`, and `last_checked` timestamp. |

### 3.2 Dataclass: `RawPayload`

Location: `src/hydra/adapters/base.py`

| Field | Type | Description |
|---|---|---|
| `stream_id` | `str` | Source stream identifier. |
| `fetched_at` | `datetime` (UTC) | Timestamp of fetch completion. |
| `content` | `bytes` | Raw response body. |
| `content_type` | `str` | MIME type from response headers. |
| `http_status` | `int` | HTTP status code. |
| `headers` | `dict[str, str]` | Response headers (selected: rate-limit, etag, last-modified). |
| `raw_hash` | `str` | SHA-256 of `content`, computed via `compute_raw_hash`. |

### 3.3 Dataclass: `AdapterHealth`

Location: `src/hydra/adapters/base.py`

| Field | Type | Description |
|---|---|---|
| `stream_id` | `str` | Source stream identifier. |
| `status` | `Enum[OK, DEGRADED, UNREACHABLE]` | Health state. |
| `latency_ms` | `float` | Round-trip time of probe. |
| `last_checked` | `datetime` (UTC) | Timestamp of check. |
| `detail` | `str | None` | Optional diagnostic message. |

### 3.4 Exception Hierarchy

Location: `src/hydra/adapters/exceptions.py`

- `AdapterError` — base exception for all adapter failures.
  - `FetchError` — upstream unreachable, timeout, or non-retryable HTTP error.
  - `RateLimitError(FetchError)` — HTTP 429 or equivalent. Must carry `retry_after` seconds.
  - `ParseError` — response body cannot be parsed into expected structure.
  - `ValidationError` — record fails validation rules.
  - `AdapterRegistryMismatch` — stream registry declares a different adapter type than the instantiated subclass.

### 3.5 Retry Policy

Implemented in `BaseAdapter.fetch` wrapper (not in subclass `fetch` directly):
- Exponential backoff: initial 1s, factor 2, max 5 retries, ceiling 60s.
- On `RateLimitError`: honor `retry_after` from upstream header, minimum 1s.
- On 5xx: retry. On 4xx (except 429): do not retry, raise `FetchError`.
- All retries logged at INFO with attempt count.

### 3.6 Structured Logging Contract

All adapters log via `structlog` with these guaranteed fields on every log event:
- `stream_id`, `adapter_type`, `event` (fetch_start, fetch_complete, parse_complete, validate_complete, normalize_complete, run_complete, error), `duration_ms`, `record_count` (where applicable), `error_detail` (on failure).

## 4. Generic REST/JSON Adapter — `src/hydra/adapters/rest_json.py`

### 4.1 Class: `RestJsonAdapter(BaseAdapter)`

The workhorse adapter. Handles any source that exposes a REST API returning JSON (or GeoJSON, JSON-LD). Covers ~84 streams across tiers 1–8, 13, 15–16, 19–20, 23, 25–27.

### 4.2 `fetch` Implementation

- Reads `base_url`, `endpoint_path`, `query_params`, and `auth_pattern` from the stream registry entry.
- Constructs the full URL. Applies query parameters including any dynamic parameters (e.g., `starttime={last_fetch_time}`).
- Auth dispatch based on `auth_pattern` field:
  - `none` — no auth headers.
  - `api_key` — reads key name and value from `HydraSettings.credentials[stream_id]`, injects as header or query param per registry `auth_key_location` field.
- Uses `aiohttp.ClientSession` with configurable timeout from `HydraSettings.http_timeout_seconds` (default: 30).
- Supports pagination: if registry entry declares `pagination_type` (one of: `offset`, `cursor`, `link_header`, `page_number`), the adapter follows pages until exhaustion or `max_pages` limit (from registry, default 10). Accumulates all pages into a single `RawPayload` with concatenated content.
- Sets `User-Agent` header to `HYDRA/{version}` from settings.
- ETag/Last-Modified caching: if the stream registry entry sets `supports_conditional: true`, the adapter stores and sends `If-None-Match` / `If-Modified-Since` headers. On HTTP 304, returns an empty `RawPayload` (content = empty bytes, http_status = 304). The `run` method short-circuits on 304 — returns empty list, logs at DEBUG.

### 4.3 `parse` Implementation

- Decodes `RawPayload.content` as UTF-8 JSON.
- If the registry entry declares `response_root_path` (a dot-delimited JSON path, e.g., `features` or `data.results`), traverses to that key and extracts the list of records.
- If no `response_root_path`, treats the top-level response as either a list (use directly) or a single object (wrap in list).
- GeoJSON handling: if `content_type` contains `geo+json` or registry declares `format: geojson`, extracts from `features` array automatically.
- Each extracted dict is preserved as-is — no transformation at this stage.

### 4.4 `validate` Implementation

- Applies required-field checks: the registry entry declares `required_fields` (list of dot-paths). Any record missing a required field is dropped and logged.
- Type coercion: if registry declares `field_types` mapping (e.g., `{"magnitude": "float", "time": "iso8601"}`), attempts coercion. Records failing coercion are dropped and logged.
- Deduplication within batch: uses `compute_raw_hash` on each record's JSON serialization. Drops intra-batch duplicates.

### 4.5 `normalize` (inherited from base)

Uses the `field_mapping` declared in the stream registry entry to map source fields to `NormalizedRecord` fields. The mapping is a dict of `{normalized_field: source_dot_path}`. Unmapped `NormalizedRecord` fields are set to `None`. The `raw_hash` field is populated from the per-record hash computed during validation.

### 4.6 Configuration-Driven Design

The REST/JSON adapter is deliberately generic. Stream-specific behavior is encoded entirely in `stream_registry.yaml` entries — not in adapter code. Adding a new REST/JSON source requires only a new registry entry, zero code changes.

## 5. Stream Registry Entry Schema (REST/JSON specific fields)

These fields extend the base registry schema established in P0, applicable when `adapter_type: rest_json`:

| Field | Type | Required | Description |
|---|---|---|---|
| `base_url` | `str` | yes | Root URL of the API. |
| `endpoint_path` | `str` | yes | Path appended to base_url. |
| `query_params` | `dict` | no | Static query parameters. |
| `dynamic_params` | `dict` | no | Parameters with template variables (e.g., `{last_fetch_time}`). |
| `auth_pattern` | `enum[none, api_key]` | yes | Authentication strategy. |
| `auth_key_location` | `enum[header, query]` | no | Where to inject API key. Required if `auth_pattern: api_key`. |
| `auth_key_name` | `str` | no | Header name or query param name for the key. |
| `response_root_path` | `str` | no | Dot-delimited path to the records array in the response. |
| `required_fields` | `list[str]` | no | Dot-paths of fields that must be present. |
| `field_types` | `dict[str, str]` | no | Expected types for coercion during validation. |
| `field_mapping` | `dict[str, str]` | yes | Maps NormalizedRecord fields to source dot-paths. |
| `pagination_type` | `enum[offset, cursor, link_header, page_number]` | no | Pagination strategy. |
| `pagination_param` | `str` | no | Query param name for pagination (e.g., `offset`, `cursor`, `page`). |
| `pagination_limit` | `int` | no | Records per page. |
| `max_pages` | `int` | no | Safety ceiling on page follows. Default: 10. |
| `supports_conditional` | `bool` | no | Whether source supports ETag/Last-Modified. Default: false. |
| `format` | `enum[json, geojson, json-ld]` | no | Response format hint. Default: json. |

## 6. Test Specification

### 6.1 `tests/test_base_adapter.py`

- Test that `BaseAdapter` cannot be instantiated directly (abstract).
- Test that a minimal concrete subclass must implement `fetch`, `parse`, `validate`.
- Test that `normalize` correctly maps fields using a mock `field_mapping`.
- Test that `run` calls methods in correct order and returns `list[NormalizedRecord]`.
- Test that `health_check` returns `AdapterHealth` with correct status enum values.
- Test retry policy: mock a sequence of 503 → 503 → 200 and verify 3 attempts with backoff.
- Test `RateLimitError` handling: mock 429 with `Retry-After: 5` header, verify adapter waits.
- Test `AdapterRegistryMismatch` raised when stream registry declares wrong adapter type.

### 6.2 `tests/test_rest_json_adapter.py`

- Test basic fetch + parse + validate + normalize pipeline with a mocked USGS earthquake GeoJSON response.
- Test pagination: mock a 3-page offset-paginated API, verify all pages accumulated.
- Test cursor pagination: mock cursor-based API, verify cursor following.
- Test ETag caching: first request returns 200 + ETag, second request returns 304, verify short-circuit.
- Test required-field validation: inject records with missing required fields, verify they are dropped.
- Test type coercion: inject records with string-typed numbers, verify coercion to float/int.
- Test deduplication: inject duplicate records within a batch, verify only unique records pass.
- Test auth injection: verify API key appears in correct location (header vs query param).
- Test configuration-driven behavior: two different stream registry entries produce different fetch URLs and parse paths without code changes.
