## 1. Purpose

Deliver two specialized adapters:
- **TAP/VO Adapter** — covers IVOA Table Access Protocol archives for astronomy and astrophysics. Targets: NASA Exoplanet Archive, MAST, HEASARC, Gaia DR3. Tier 22.
- **S3/Bulk Adapter** — covers large-scale dataset ingestion from S3-compatible object stores and bulk download endpoints. Targets: JWST, TESS, NOAA SWPC, Kaguya, Common Crawl. Tiers 3, 4, 17, 22, 24.

Both subclass `BaseAdapter` from P1 and produce `NormalizedRecord` output.

## 3. TAP/VO Adapter — `src/hydra/adapters/tap_vo.py`

### 3.1 Class: `TapVoAdapter(BaseAdapter)`

The International Virtual Observatory Alliance (IVOA) Table Access Protocol (TAP) defines a standardized interface for querying astronomical databases using ADQL (Astronomical Data Query Language). All compliant archives expose `/tap/sync` and `/tap/async` endpoints with identical semantics.

### 3.2 TAP Query Modes

Two modes, selected via registry `tap_mode` field:

| Mode | Endpoint | Behavior | Use Case |
|---|---|---|---|
| `sync` | `/tap/sync` | Blocking request. Server executes query and returns results in the HTTP response body. | Small-to-medium queries. Fast turnaround. |
| `async` | `/tap/async` | Non-blocking. Server creates a job, returns a job URL. Client polls for completion, then retrieves results. | Large queries. Long-running. Required for Gaia DR3 bulk queries. |

### 3.3 `fetch` Implementation

- Reads `base_url` (e.g., `https://exoplanetarchive.ipac.caltech.edu`), `tap_endpoint` (e.g., `/TAP/sync`), and `tap_mode` from the stream registry entry.
- Constructs the ADQL query from the registry-declared `adql_template` field. The template supports variable substitution using Python `str.format_map` with the following variables:
  - `{last_fetch_time}` — ISO 8601 timestamp of last successful fetch, provided by the scheduler.
  - `{table_name}` — target table, declared in registry.
  - `{max_rows}` — row limit, declared in registry (default: 10000).
  - `{custom_where}` — optional additional WHERE clause from registry.
- Request parameters sent as POST form data:
  - `REQUEST=doQuery`
  - `LANG=ADQL`
  - `QUERY={resolved_adql}`
  - `FORMAT={response_format}` — from registry, one of: `votable`, `csv`, `fits`. Default: `votable`.

**Sync mode:**
- Single POST to `{base_url}{tap_endpoint}`.
- Response body is the query result in the declared format.
- Timeout from `HydraSettings.http_timeout_seconds` (default: 30). For TAP queries, the registry may override with `tap_timeout_seconds` (default: 120) to accommodate larger queries.

**Async mode:**
- Phase 1 — Job creation: POST to `{base_url}/tap/async` with query parameters. Server returns HTTP 303 with `Location` header pointing to the job URL, or HTTP 200 with job URL in response body.
- Phase 2 — Poll for completion: GET `{job_url}/phase` repeatedly. Possible phases: `PENDING`, `QUEUED`, `EXECUTING`, `COMPLETED`, `ERROR`, `ABORTED`. Poll interval from registry `async_poll_interval_seconds` (default: 5). Maximum poll duration from registry `async_max_wait_seconds` (default: 600).
- Phase 3 — Retrieve results: on `COMPLETED`, GET `{job_url}/results/result`. Response body is the query result.
- On `ERROR`: GET `{job_url}/error` for error message. Raise `FetchError` with the server-provided error detail.
- On poll timeout (max wait exceeded): attempt to DELETE the job (`DELETE {job_url}`), then raise `FetchError` with timeout detail.

**Auth dispatch:**
- `none` — most TAP services are open (NASA Exoplanet Archive, HEASARC, MAST).
- `account_token` — Gaia DR3 and some ESA services require an account token. Read from `HydraSettings.credentials[stream_id]`, inject as cookie or `Authorization` header per registry `auth_token_location` field.

**Response format handling:**
- `votable` — `application/x-votable+xml`. Primary format. XML-based.
- `csv` — `text/csv`. Fallback for services with limited VOTable support.
- `fits` — `application/fits`. Binary. Used for large catalog queries.

### 3.4 `parse` Implementation

**VOTable parsing:**
- Parse using `astropy.io.votable`. Load the VOTable from bytes via `astropy.io.votable.parse` with `verify='warn'` to handle minor schema deviations common across TAP services.
- Extract the first `TABLE` element from the first `RESOURCE`.
- Iterate rows. Each row becomes a dict keyed by column names from the `FIELD` definitions.
- Handle VOTable-specific null representations: empty strings, NaN markers, and the `null` attribute on `VALUES` elements. Map all to Python `None`.
- Preserve column metadata (unit, UCD, utype, datatype) in a sidecar dict attached to the `RawPayload` headers for downstream use by the normalize step.

**CSV parsing:**
- Standard CSV parsing via Python `csv.DictReader`.
- First row as headers. Strip whitespace from headers and values.

**FITS parsing:**
- Parse using `astropy.io.fits`. Open from bytes via `io.BytesIO`.
- Extract the first binary table extension (`BinTableHDU`).
- Convert to `astropy.table.Table`, then iterate rows to produce dicts.
- Handle FITS-specific null values (`TNULL` keyword) — map to Python `None`.

**Common post-parse step:**
- Each parsed record is tagged with `tap_service_url` (the full URL queried) and `adql_query` (the resolved query string) for provenance tracking.

### 3.5 `validate` Implementation

- Required-field validation per registry `required_fields`.
- Coordinate validation where applicable: `ra` (right ascension) in [0, 360], `dec` (declination) in [-90, 90]. Field names for coordinates declared in registry `coordinate_fields` mapping (e.g., `{"ra": "ra", "dec": "dec"}`). If not declared, coordinate validation is skipped.
- Numeric range validation: registry may declare `numeric_ranges` as a dict of `{field: [min, max]}`. Records with values outside declared ranges are dropped and logged.
- Type coercion: VOTable columns carry datatype metadata. The adapter coerces string representations to native Python types (int, float, str) based on VOTable `FIELD` datatype attributes. Coercion failures are logged at WARNING and the record is dropped.
- Deduplication within batch: uses a composite key declared in registry `dedup_key_fields` (e.g., `["pl_name", "disc_year"]` for exoplanets, `["obsid"]` for MAST observations). Falls back to `compute_raw_hash` on the full record if no `dedup_key_fields` declared.

### 3.6 Stream Registry Entry Schema (TAP/VO-specific fields)

| Field | Type | Required | Description |
|---|---|---|---|
| `tap_mode` | `enum[sync, async]` | yes | Query execution mode. |
| `tap_endpoint` | `str` | no | Path to TAP endpoint. Default: `/tap/sync` for sync, `/tap/async` for async. |
| `adql_template` | `str` | yes | ADQL query template with `{variable}` placeholders. |
| `table_name` | `str` | yes | Target table name for ADQL query. |
| `max_rows` | `int` | no | Row limit for query. Default: 10000. |
| `custom_where` | `str` | no | Additional WHERE clause fragment appended to ADQL template. |
| `response_format` | `enum[votable, csv, fits]` | no | Desired response format. Default: `votable`. |
| `tap_timeout_seconds` | `int` | no | HTTP timeout override for TAP queries. Default: 120. |
| `async_poll_interval_seconds` | `int` | no | Seconds between async job phase polls. Default: 5. |
| `async_max_wait_seconds` | `int` | no | Maximum seconds to wait for async job completion. Default: 600. |
| `auth_token_location` | `enum[cookie, header]` | no | Where to inject account token. Required if `auth_pattern: account_token`. |
| `coordinate_fields` | `dict[str, str]` | no | Maps `ra` and `dec` to source field names for coordinate validation. |
| `numeric_ranges` | `dict[str, list[float]]` | no | Field-level numeric range constraints as `{field: [min, max]}`. |
| `dedup_key_fields` | `list[str]` | no | Fields forming the composite deduplication key. |

## 4. S3/Bulk Adapter — `src/hydra/adapters/s3_bulk.py`

### 4.1 Class: `S3BulkAdapter(BaseAdapter)`

Handles large-scale dataset ingestion from S3-compatible object stores (AWS public buckets, Earthdata Cloud, STAC catalogs) and bulk HTTP download endpoints. Designed for datasets measured in gigabytes to terabytes where individual object retrieval and differential sync are essential.

### 4.2 Ingestion Modes

Three modes, selected via registry `bulk_mode` field:

| Mode | Mechanism | Use Case |
|---|---|---|
| `s3_listing` | List objects in an S3 bucket/prefix, selectively download new or changed objects. | JWST, TESS, NOAA SWPC, Kaguya (public S3 buckets). |
| `manifest` | Fetch a manifest file (JSON, CSV, or text listing) that enumerates available objects with metadata (size, checksum, last-modified). Download only objects not present in local tracking state. | Common Crawl (WARC manifest), Earthdata bulk downloads. |
| `http_bulk` | Sequential or parallel HTTP GET of files from a list of URLs declared in registry or discovered via a directory page. | Bulk download endpoints without S3 access. |

### 4.3 `fetch` Implementation

**S3 listing mode (`s3_listing`):**
- Uses `boto3` S3 client in anonymous mode (`UNSIGNED` signature) for public buckets. For authenticated buckets, reads credentials from `HydraSettings.credentials[stream_id]` with keys `aws_access_key_id`, `aws_secret_access_key`, and optional `aws_session_token`.
- Reads `s3_bucket`, `s3_prefix`, and optional `s3_region` from registry.
- Lists objects under the prefix using `list_objects_v2` with pagination (continuation tokens).
- Applies file extension filter from registry `file_extension_filter` (e.g., `[".fits", ".json", ".csv"]`). Objects not matching any declared extension are skipped.
- Applies date filter: only objects with `LastModified` after `{last_fetch_time}` are candidates for download. This is the primary differential sync mechanism.
- For each candidate object, checks against a local manifest tracking file (JSON, stored at a path derived from `stream_id`). The manifest records `{object_key: {etag, size, last_modified, fetched_at}}`. Objects already present with matching `ETag` are skipped.
- Downloads selected objects using `get_object`. For objects larger than `streaming_threshold_bytes` (from registry, default: 50 MB), uses streaming download — reads in chunks of `chunk_size_bytes` (default: 8 MB) and writes to a temporary file rather than holding in memory. For objects at or below the threshold, reads into memory as bytes.
- Each downloaded object produces one `RawPayload` with `content` set to the bytes (for in-memory) or a sentinel value indicating temp file path (for streamed). The `headers` dict includes `s3_key`, `s3_etag`, `s3_size`, and `s3_last_modified`.
- After successful download, updates the local manifest tracking file.
- Rate limiting: configurable `max_concurrent_downloads` (from registry, default: 5) using `asyncio.Semaphore`. Configurable `download_delay_seconds` (default: 0) between sequential downloads.

**Manifest mode (`manifest`):**
- Fetches the manifest file from `manifest_url` declared in registry. Manifest format declared in `manifest_format` (one of: `json`, `csv`, `text_lines`).
- Parses the manifest to extract a list of object entries, each with at minimum a URL or key, and optionally size, checksum, and last-modified.
- Applies differential sync against the local tracking manifest (same mechanism as S3 listing mode, keyed by URL or object key).
- Downloads new/changed objects via HTTP GET (not S3 API). Streaming and concurrency controls identical to S3 listing mode.

**HTTP bulk mode (`http_bulk`):**
- Reads `bulk_urls` (static list) or `bulk_index_url` (a page to scrape for download links) from registry.
- If `bulk_index_url` is declared, fetches the index page and extracts download links matching `link_pattern` (regex from registry).
- Applies file extension filter and differential sync against local tracking manifest.
- Downloads via HTTP GET with streaming and concurrency controls.

**Auth dispatch:**
- `none` — public S3 buckets (JWST, TESS, NOAA SWPC, Kaguya).
- `aws_credentials` — private S3 buckets. Credentials from `HydraSettings.credentials[stream_id]`.
- `account_token` — Earthdata Login OAuth. The adapter implements the Earthdata OAuth2 flow: redirect to `urs.earthdata.nasa.gov/oauth/authorize`, exchange code for token, use bearer token for subsequent requests. Token refresh handled automatically. Credentials (client_id, client_secret, username, password) from `HydraSettings.credentials[stream_id]`.
- `api_key` — same as REST/JSON adapter, for bulk endpoints requiring API key.

### 4.4 `parse` Implementation

Parser dispatch based on file extension (from the S3 key or URL) or `content_type` from response headers. The registry may also declare `parser_override` to force a specific parser regardless of extension.

| Extension / Content-Type | Parser | Library | Output |
|---|---|---|---|
| `.json`, `application/json` | JSON parser | stdlib `json` | List of dicts (if array) or single dict wrapped in list. |
| `.csv`, `text/csv` | CSV parser | stdlib `csv.DictReader` | List of dicts, first row as headers. |
| `.fits`, `application/fits` | FITS parser | `astropy.io.fits` | Extract binary table extensions. Each row becomes a dict. Header metadata preserved as sidecar. |
| `.nc`, `.nc4`, `application/x-netcdf` | NetCDF parser | `xarray` + `netCDF4` | Open dataset, extract declared variables (from registry `netcdf_variables` list) as flattened dicts. Each time step or spatial point becomes one record depending on registry `netcdf_flatten_dim`. |
| `.hdf5`, `.h5`, `application/x-hdf5` | HDF5 parser | `h5py` | Navigate to declared dataset path (from registry `hdf5_dataset_path`). Extract as array, convert rows to dicts using declared column names from registry `hdf5_columns`. |
| `.tif`, `.tiff`, `image/tiff` | GeoTIFF parser | Metadata extraction only | Extract geospatial metadata (bounds, CRS, resolution, band count) from headers. Raw raster data stored as-is for downstream processing. Each file becomes one metadata dict. |
| `.mseed`, `application/vnd.fdsn.mseed` | miniSEED parser | Header-only extraction | Identical to FDSN adapter behavior: extract trace-level metadata (network, station, channel, location, starttime, endtime, sample_rate, num_samples) from headers. Raw binary preserved for S3 storage. |
| `.warc`, `.warc.gz` | WARC parser | Metadata extraction only | Extract WARC record headers (WARC-Type, WARC-Target-URI, WARC-Date, Content-Length). Do not decompress or parse payload at ingestion. Each WARC record header block becomes one dict. |

**Common post-parse step:**
- Each parsed record is tagged with `source_object_key` (S3 key or URL), `source_bucket` (if S3), and `object_size_bytes` for provenance.

### 4.5 `validate` Implementation

- Required-field validation per registry `required_fields`.
- Format-specific validation:
  - FITS: validate that expected HDU extensions are present (from registry `expected_hdu_names`).
  - NetCDF: validate that declared variables exist in the dataset.
  - HDF5: validate that declared dataset path exists.
  - GeoTIFF: validate CRS is parseable, bounds are within valid geographic ranges.
- Checksum validation: if the manifest or S3 metadata provides a checksum (MD5, SHA-256), verify against the downloaded content. Mismatches are logged at ERROR and the object is rejected.
- Size validation: if expected size is known from manifest/S3 metadata, verify downloaded size matches. Mismatches are logged at ERROR and the object is rejected.
- Deduplication: uses `compute_raw_hash` on the raw content bytes. Objects with hashes already present in the local tracking manifest are skipped.

### 4.6 Manifest Tracking State

The local manifest tracking file is a JSON file stored at `{HydraSettings.data_dir}/manifests/{stream_id}_manifest.json`. Structure:

```
{
  "stream_id": "jwst_public",
  "last_sync": "2026-04-03T00:00:00Z",
  "objects": {
    "s3_key_or_url": {
      "etag": "abc123",
      "size": 1048576,
      "last_modified": "2026-04-02T12:00:00Z",
      "raw_hash": "sha256:...",
      "fetched_at": "2026-04-03T00:05:00Z",
      "status": "ingested"
    }
  }
}
```

The manifest is loaded at the start of `fetch`, consulted during differential sync, and updated after each successful download. File-level locking (via `fcntl` or equivalent) prevents concurrent adapter instances from corrupting the manifest.

|---|---|---|---|
| `bulk_mode` | `enum[s3_listing, manifest, http_bulk]` | yes | Ingestion mode. |

| `s3_prefix` | `str` | no | S3 key prefix to scope listing. Default: empty (bucket root). |
| `s3_region` | `str` | no | AWS region for the bucket. Default: `us-east-1`. |

### 6.1 `tests/test_tap_vo_adapter.py`

- **Sync VOTable query:** Mock a TAP sync endpoint returning a small VOTable (use `tests/fixtures/sample.vot`). Verify parse extracts correct number of rows with correct column names and values. Verify null handling (VOTable empty strings → Python `None`).
