## 1. Purpose

Build the intelligence product generation layer that transforms raw records and correlation results into structured analytical outputs. P10 owns:

1. **Product framework** — pluggable intelligence product generators that consume `NormalizedRecord` and `CorrelationResult` data to produce structured reports.
2. **Three MVP products** — Situation Report (SITREP), Entity Dossier, and Threat Assessment — covering the primary analytical use cases.
3. **Query layer** — unified read interface across PostgreSQL, Elasticsearch, and Neo4j for analytical queries.
4. **Graph traversal** — Neo4j-backed network analysis for entity relationship mapping.
5. **Timeline construction** — temporal event sequencing across tiers for chronological analysis.
6. **Scheduled generation** — Airflow DAGs for periodic product generation with on-demand capability via P11.

After P10, HYDRA produces consumable intelligence — not just correlated data. Analysts receive structured products that synthesize cross-tier information into actionable assessments.

## 3. Architecture

### 3.1 Data Flow

```
Product Request (scheduled DAG or on-demand via P11)
       │
       ▼
AnalysisEngine.generate(product_type, params)
       │
       ├── 1. QueryLayer: fetch relevant records from PG/ES/InfluxDB
       ├── 2. QueryLayer: fetch relevant correlations from PG
       ├── 3. GraphAnalyzer: traverse Neo4j for entity networks (if applicable)
       ├── 4. TimelineBuilder: construct event timeline (if applicable)
       ├── 5. ProductGenerator.generate(data_bundle) → IntelligenceProduct
       │       ├── Synthesize records + correlations + graph + timeline
       │       ├── Apply analytical templates
       │       └── Score confidence and completeness
       ├── 6. Persist product to PostgreSQL (intelligence_products table)
       └── 7. Return product
```

```python
class AnalysisEngine:
    """
    Orchestrates intelligence product generation.

    Responsibilities:
    1. Accept product generation requests (type + parameters).
    2. Assemble DataBundle via QueryLayer, GraphAnalyzer, TimelineBuilder.
    3. Dispatch to the appropriate product generator.
    4. Persist the generated product.
    5. Return the product.
    """

    def __init__(
        self,
        query_layer: QueryLayer,
        graph_analyzer: GraphAnalyzer,
        timeline_builder: TimelineBuilder,
        pg_engine: PostgresEngine,
        settings: HydraSettings,
    ):
        self._queries = query_layer
        self._graph = graph_analyzer
        self._timeline = timeline_builder
        self._pg = pg_engine
        self._settings = settings
        self._products: dict[str, BaseProduct] = {}

    def register_product(self, product: BaseProduct) -> None:
        """Register a product generator by its type ID."""
        self._products[product.product_type] = product

    async def generate(
        self,
        product_type: str,
        params: ProductParams,
    ) -> IntelligenceProduct:
        """
        Generate an intelligence product.

        Steps:
        1. Resolve product generator.
        2. Build DataBundle:
           a. Query records from PG/ES per product's source tiers and time window.
           b. Query correlations involving those records.
           c. If product requires graph: run GraphAnalyzer.
           d. If product requires timeline: run TimelineBuilder.
        3. Call generator.generate(bundle).
        4. Compute product_hash for dedup.
        5. Persist to intelligence_products table.
        6. Return product.
        """
        ...

    async def _build_data_bundle(
        self,
        product: BaseProduct,
        params: ProductParams,
    ) -> DataBundle:
        """
        Assemble all data needed for product generation.

        Queries are parallelized where possible:
        - Records and correlations can be fetched concurrently.
        - Graph and timeline depend on record results.
        """
        ...

    async def _persist_product(
        self, product: IntelligenceProduct
    ) -> None:
        """
        Write product to intelligence_products table.

        Dedup: if product_hash already exists, update rather than insert

        """
        ...

    async def list_products(
```

## 7. MVP Products

### 7.1 Situation Report (SITREP)

Location: `src/hydra/analysis/products/situation_report.py`

**Purpose:** Periodic overview of significant events across monitored tiers within a time window. The primary "what happened" product — a structured digest of ingested intelligence.

**Source tiers:** All 28 (filtered by significance). Default lookback: 24 hours.

```python
class SituationReport(BaseProduct):
    product_type = "situation_report"
    source_tiers = list(range(1, 29))  # all tiers
    requires_graph = False
    requires_timeline = True
    default_lookback_hours = 24.0

    def __init__(self, settings: HydraSettings):
        self._max_events_per_tier = settings.analysis.sitrep_max_events_per_tier  # default: 20
        self._significance_threshold = settings.analysis.sitrep_significance_threshold  # default: 0.3

    async def generate(
        self, bundle: DataBundle, params: ProductParams
    ) -> IntelligenceProduct:
        """
        SITREP generation algorithm:

        1. Score record significance:
           - Records involved in correlations score higher.
           - Records from high-frequency tiers with anomalous values score higher.
           - Records with high confidence score higher.
           - Formula: significance = (0.4 * correlation_factor) +
                                     (0.3 * anomaly_factor) +
                                     (0.3 * confidence)

        2. Filter to records above significance_threshold.

        3. Group by tier, take top N per tier by significance.

        4. Construct sections:
           a. "Executive Summary" — narrative overview of the period.
           b. "Key Developments" — top 10 most significant events across all tiers.
           c. Per-tier sections — grouped by domain:
              - "Geophysical & Environmental" (Tiers 1, 2, 3, 24, 25)
              - "Security & Conflict" (Tiers 6, 15, 16, 19, 20)
              - "Economic & Governance" (Tiers 5, 8, 9, 10, 11, 12, 13)
              - "Health & Human Rights" (Tiers 7, 21, 26)
              - "Space & Science" (Tiers 4, 22, 23)
              - "Infrastructure & Energy" (Tiers 18, 27)
              - "Open Source & Social" (Tiers 14, 17, 28)
           d. "Cross-Tier Correlations" — significant correlations from the period.
           e. "Timeline" — chronological event sequence from TimelineBuilder.

        5. Generate key_findings from top-significance events.

        6. Score product:
           - confidence: weighted average of source record confidences.
           - completeness: (tiers with data / total source tiers).
        """
        ...
```

**Significance scoring detail:**

```python
def _score_significance(
    self,
    record: NormalizedRecord,
    correlation_map: dict[str, list[CorrelationResult]],
) -> float:
    """
    Score a record's significance for SITREP inclusion.

    correlation_factor: 0.0 if no correlations, scales to 1.0 based on
        count and max confidence of correlations involving this record.
    anomaly_factor: 0.0 for baseline records. Elevated for:
        - Geophysical: magnitude > 4.0 or depth < 10km
        - Cyber: CVSS > 7.0 or KEV-listed
        - Conflict: fatalities > 0
        - Maritime/Aviation: speed anomaly or position in restricted zone
        - Default: 0.5 (no anomaly detection for this tier yet)
    confidence: record.confidence directly.
    """
    ...
```

### 7.2 Entity Dossier

Location: `src/hydra/analysis/products/entity_dossier.py`

**Purpose:** Comprehensive profile of a specific entity (person, organization, vessel, threat actor) assembled from all tiers where the entity appears. The primary "who/what is this" product.

**Source tiers:** 8, 14, 15, 16, 19, 21 (entity-rich tiers). Expands dynamically based on graph traversal.

```python
class EntityDossier(BaseProduct):
    product_type = "entity_dossier"
    source_tiers = [8, 14, 15, 16, 19, 21]
    requires_graph = True
    requires_timeline = True
    default_lookback_hours = 8760.0  # 365 days — dossiers are comprehensive

    def __init__(self, settings: HydraSettings, es_engine: ElasticsearchEngine | None):
        self._es = es_engine
        self._max_network_depth = settings.analysis.dossier_network_depth  # default: 2
        self._max_network_nodes = settings.analysis.dossier_max_network_nodes  # default: 50

    async def generate(
        self, bundle: DataBundle, params: ProductParams
    ) -> IntelligenceProduct:
        """
        Entity Dossier generation algorithm:

        Requires params.entity_id or params.entity_name.

        1. Entity resolution:
           a. If entity_id provided: exact match in PG payload fields
              using ENTITY_EXTRACTION_MAP from P9.
           b. If entity_name provided: fuzzy search via Elasticsearch
              more_like_this + PG payload JSONB containment query.
           c. Collect all records referencing this entity across all source tiers.

        2. Graph network extraction:
           a. Find entity's Neo4j node(s) — may have multiple nodes
              across tiers (Tier 16 STIX node, Tier 19 sanctions node, etc.).
           b. Traverse outward to max_network_depth hops.
           c. Cap at max_network_nodes.
           d. Compute centrality metrics for the entity within its network.
           e. Detect communities — which cluster does this entity belong to?

        3. Timeline construction:
           a. All records referencing this entity, ordered chronologically.
           b. Annotate with correlation links.

        4. Construct sections:
           a. "Entity Profile" — name, aliases, identifiers, type, first/last seen.
           b. "Sanctions & Regulatory Status" — Tier 19 records (OFAC, EU, UN listings).
           c. "Conflict & Event Involvement" — Tier 15 records (ACLED, GDELT events).
           d. "Cyber Threat Activity" — Tier 16 records (STIX objects, ATT&CK mappings).
           e. "Human Rights Concerns" — Tier 21 records.
           f. "International Organization References" — Tier 8 records.
           g. "Arms & Defense Connections" — Tier 14 records.
           h. "Network Analysis" — graph summary: key connections, centrality,
              community membership, shortest paths to other entities of interest.
           i. "Activity Timeline" — chronological event sequence.
           j. "Cross-Tier Correlations" — correlations involving this entity.

        5. Sections are only included if data exists — empty tiers are omitted.

        6. Score product:
           - confidence: weighted average of source record confidences,
             boosted by exact ID matches (vs. fuzzy name matches).
           - completeness: (tiers with entity data / total entity-relevant tiers).
        """
        ...
```

**Entity resolution strategy:**

```python
async def _resolve_entity(
    self, params: ProductParams, bundle: DataBundle
) -> EntityResolution:
    """
    Multi-strategy entity resolution.

    Phase 1 — Exact ID match:
    Query PG: payload @> '{"entity_id": "<id>"}' across source tiers.
    Also check: stix_id, ofac_id, actor_id, case_id per ENTITY_EXTRACTION_MAP.

    Phase 2 — Fuzzy name match (if Phase 1 yields < 3 records or entity_name provided):
    Query ES: more_like_this on entity name across text-heavy tier indices.
    Filter by Jaro-Winkler similarity >= 0.80.

    Phase 3 — Graph expansion:
    From matched records, find Neo4j nodes.
    Traverse 1 hop to discover additional records not caught by text search.

    Returns EntityResolution with:
    - canonical_name: most frequent name across matches
    - identifiers: all discovered IDs (STIX, OFAC, ACLED, etc.)
    - records: all matched NormalizedRecords
    - match_quality: "exact_id" | "fuzzy_name" | "graph_expanded"
    """
    ...
```

### 7.3 Threat Assessment

Location: `src/hydra/analysis/products/threat_assessment.py`

**Purpose:** Focused analysis of a specific threat vector, region, or scenario synthesizing indicators from multiple domains. The primary "what could happen" product — forward-looking assessment based on converging signals.

**Source tiers:** 6, 15, 16, 19, 20, 27 (threat-relevant tiers, matching P9 threat_convergence pipeline).

```python
class ThreatAssessment(BaseProduct):
    product_type = "threat_assessment"
    source_tiers = [6, 15, 16, 19, 20, 27]
    requires_graph = True
    requires_timeline = True
    default_lookback_hours = 168.0  # 7 days

    def __init__(self, settings: HydraSettings):
        self._threat_level_thresholds = settings.analysis.threat_level_thresholds
        self._min_convergence_tiers = settings.analysis.threat_min_convergence_tiers  # default: 2

    async def generate(
        self, bundle: DataBundle, params: ProductParams
    ) -> IntelligenceProduct:
        """
        Threat Assessment generation algorithm:

        1. Scope definition:
           - If params.region: filter all records to that geographic region.
           - If params.keywords: filter by keyword presence in payload.
           - If neither: assess all threat-relevant records in the time window.

        2. Signal aggregation:
           a. Group records by geographic region (country-level).
           b. Within each region, count signals per tier.
           c. Identify regions with multi-tier convergence
              (signals from >= min_convergence_tiers different tiers).

        3. Threat level scoring per region:
           - LOW: 1 tier reporting, low-confidence correlations
           - MODERATE: 2 tiers reporting, or 1 tier with high-confidence correlations
           - HIGH: 3+ tiers reporting with medium+ confidence correlations
           - CRITICAL: 4+ tiers reporting with high-confidence correlations
             and temporal clustering (events within 24h)

           Thresholds configurable via threat_level_thresholds.

        4. Construct sections:
           a. "Threat Overview" — narrative summary of assessed threats.
           b. "Threat Matrix" — table: region × tier with signal counts
              and threat levels.
           c. "Critical Threats" — detailed analysis of CRITICAL/HIGH regions:
              - Contributing indicators per tier
              - Correlation evidence
              - Entity involvement (from graph)
              - Temporal pattern (from timeline)
           d. "Emerging Patterns" — regions at MODERATE that show
              escalation indicators (increasing signal frequency,
              new tier involvement).
           e. "Indicator Timeline" — chronological sequence of threat indicators.
           f. "Network Analysis" — graph summary of threat actor networks
              active in assessed regions.
           g. "Assessment Confidence" — methodology disclosure:
              data sources, time window, limitations, gaps.

        5. Key findings focus on actionable assessments:
           - Which regions are at elevated threat levels and why.
           - What convergence patterns are emerging.
           - What data gaps limit assessment confidence.

        6. Score product:
           - confidence: weighted by correlation confidence and tier coverage.
           - completeness: (tiers with data / total threat-relevant tiers)
             × (regions with data / regions of interest).
        """
        ...
```

```python
class GraphAnalyzer:
    """
    Neo4j-backed network analysis for intelligence products.

    Provides graph traversal, centrality metrics, community detection,
    and shortest-path queries for entity relationship mapping.
```

### 9.2 Cypher Query Patterns

**Entity network traversal:**
```cypher
MATCH (start:Record)
WHERE start.raw_hash IN $entity_hashes
CALL apoc.path.subgraphAll(start, {
    maxLevel: $max_depth,
    limit: $max_nodes
})
YIELD nodes, relationships
RETURN nodes, relationships
```

**Fallback without APOC:**
```cypher
MATCH path = (start:Record)-[*1..$max_depth]-(connected:Record)
WHERE start.raw_hash IN $entity_hashes
WITH DISTINCT connected, min(length(path)) AS distance
ORDER BY distance
LIMIT $max_nodes
RETURN connected
```

**Degree centrality:**
```cypher
MATCH (n:Record)
WHERE n.tier IN $tiers OR $tiers IS NULL
OPTIONAL MATCH (n)-[r]-()
RETURN n.raw_hash AS node_id, n.tier AS tier, count(r) AS degree
ORDER BY degree DESC
LIMIT $top_n
```

**Shortest path:**
```cypher
MATCH path = shortestPath(
    (source:Record {raw_hash: $source_hash})-[*..5]-(target:Record {raw_hash: $target_hash})
)
RETURN path
```

```python
class TimelineBuilder:
    """
    Constructs chronological event sequences from NormalizedRecords.

    Provides temporal analysis: event ordering, clustering,
    frequency analysis, and gap detection.
    """

    def __init__(self, settings: HydraSettings):
        self._cluster_window_s = settings.analysis.timeline_cluster_window_s  # default: 3600
        self._max_events = settings.analysis.timeline_max_events  # default: 500

    async def build(
        self,
        records: dict[int, list[NormalizedRecord]],
        correlations: list[CorrelationResult],
        time_start: str,
        time_end: str,
    ) -> TimelineResult:
        """
        Build timeline from records across tiers.

        Steps:
        1. Flatten all records into TimelineEvents.
        2. Sort by timestamp.
        3. Extract event title and description from payload
           (tier-specific extraction rules).
        4. Annotate with correlation links.
        5. Score significance per event.
        6. If total events > max_events: keep top by significance.
        7. Detect temporal clusters (events within cluster_window_s).
        8. Return TimelineResult.
        """
        ...

    def _extract_event_title(
        self, record: NormalizedRecord
    ) -> str:
        """
        Tier-specific title extraction from payload.

        Tier 1: "{magnitude} M earthquake at {place}"
        Tier 15: "{event_type} in {country}: {notes}"
        Tier 16: "{type}: {name}"
        Tier 18: "{vessel_name} ({mmsi}) at {lat},{lon}"
        Tier 19: "{entity_name} — {program} listing"
        Default: "{stream_id} record at {timestamp}"
        """
        ...

    def _extract_event_description(
        self, record: NormalizedRecord
    ) -> str:
        """
        Tier-specific description extraction.

        Truncated to 500 characters. Pulls from the most
        descriptive payload field per tier.
        """
        ...

    def _detect_clusters(
        self, events: list[TimelineEvent]
    ) -> list[EventCluster]:
        """
        Group temporally proximate events into clusters.

        Algorithm: sliding window of cluster_window_s.
        Events within the window form a cluster.
        Clusters with events from 2+ tiers are flagged as
        cross-tier clusters (higher significance).

        Cluster significance = max(event significances) *
            (1.0 + 0.1 * (tier_count - 1))
        """
```

## 14. Exceptions

Location: `src/hydra/analysis/exceptions.py`

```python
class AnalysisError(Exception):
    """Base exception for analysis module."""

class ProductNotFoundError(AnalysisError):
    """Raised when a requested product_type is not registered."""
    def __init__(self, product_type: str):
        self.product_type = product_type
        super().__init__(f"Unknown product type: {product_type}")

class InsufficientDataError(AnalysisError):
    """Raised when insufficient data exists to generate a meaningful product."""
    def __init__(self, product_type: str, tiers_with_data: int, tiers_required: int):
        self.product_type = product_type
        self.tiers_with_data = tiers_with_data
        self.tiers_required = tiers_required
        super().__init__(
            f"Insufficient data for {product_type}: "
            f"{tiers_with_data}/{tiers_required} tiers have data"
        )

class EntityResolutionError(AnalysisError):
    """Raised when entity resolution fails to find any matching records."""
    def __init__(self, entity_id: str | None, entity_name: str | None):
        identifier = entity_id or entity_name or "unknown"
        super().__init__(f"Entity not found: {identifier}")

class QueryLayerError(AnalysisError):
    """Raised when an analytical query fails."""
    def __init__(self, engine: str, cause: str):
        self.engine = engine
        super().__init__(f"Query failed ({engine}): {cause}")

class GraphAnalysisError(AnalysisError):
    """Raised when Neo4j graph analysis fails."""
    def __init__(self, operation: str, cause: str):
        self.operation = operation
        super().__init__(f"Graph analysis failed ({operation}): {cause}")
```
