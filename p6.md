
# P6 — Authentication Layer

## Kiro Specification — HYDRA OSINT Platform
**Version:** 0.1.0
**Phase:** 6 (MVP)
**Depends on:** P0 (Project Scaffold & Shared Contracts), P1 (Base Adapter Interface)
**Consumed by:** All adapters (P1–P5), P7 (Storage Layer), P8 (Scheduler Engine)
**Produces:** `src/hydra/auth/__init__.py`, `src/hydra/auth/manager.py`, `src/hydra/auth/strategies.py`, `src/hydra/auth/credential_store.py`, `src/hydra/auth/exceptions.py`, `tests/test_auth_manager.py`, `tests/test_auth_strategies.py`, `tests/test_credential_store.py`

---

## 1. Purpose

Centralize all authentication logic into a single dispatch layer that every adapter delegates to. P1–P5 each implemented auth inline within their `fetch` methods — pattern-matched `if/elif` blocks reading credentials directly from `HydraSettings.credentials[stream_id]`. P6 extracts this into a unified `AuthManager` that:

1. Resolves credentials from a pluggable credential store.
2. Dispatches to the correct `AuthStrategy` based on the stream's declared `auth_pattern`.
3. Manages token lifecycle (acquisition, caching, refresh, expiry) for stateful auth patterns.
4. Provides a single `apply(request)` interface that adapters call before every outbound HTTP request.
5. Handles credential rotation and store reloading without adapter restart.

After P6, adapters no longer contain any auth logic. They call `AuthManager.apply()` and receive a fully authenticated request context.

## 2. Integration Contracts

### 2.1 Consumed from P0

- **HydraSettings** — `from hydra.config import settings` — provides `credential_store_path`, `credentials` dict, and per-stream configuration.
- **stream_registry.yaml** — each stream entry declares `auth_pattern` and pattern-specific fields (`auth_key_location`, `auth_key_name`, `auth_token_location`, `cert_path`, `key_path`).

### 2.2 Consumed from P1

- **BaseAdapter** — P6 modifies the adapter contract: `BaseAdapter.__init__` instantiates an `AuthManager` for its stream. `BaseAdapter.fetch` wrapper calls `auth_manager.apply()` before dispatching to the subclass `fetch`.
- **Exception hierarchy** — P6 extends with auth-specific exceptions (see §5).

### 2.3 Produced by P6

- **AuthManager** — single entry point for all auth operations. Adapters receive this via dependency injection.
- **AuthStrategy** (abstract) + 8 concrete strategies — one per declared auth pattern.
- **CredentialStore** (abstract) + `YamlCredentialStore` (default) — pluggable credential backend.
- **AuthenticatedRequest** — dataclass carrying headers, query params, cookies, and SSL context that adapters merge into their outbound requests.

### 4.2 Class: `AuthenticatedRequest` (dataclass)

Location: `src/hydra/auth/manager.py`

The output of every `AuthStrategy.apply()` call. Adapters merge this into their outbound HTTP request.

| Field | Type | Default | Description |
|---|---|---|---|
| `headers` | `dict[str, str]` | `{}` | Headers to add/override (e.g., `Authorization`, `X-RapidAPI-Key`). |
| `query_params` | `dict[str, str]` | `{}` | Query parameters to add (e.g., API key as query param). |
| `cookies` | `dict[str, str]` | `{}` | Cookies to set (e.g., session cookie, account token as cookie). |
| `ssl_context` | `ssl.SSLContext | None` | `None` | Custom SSL context for mutual TLS. |
| `boto3_config` | `dict[str, Any] | None` | `None` | AWS credential dict for boto3 client construction. |

### 4.3 Class: `CredentialStore` (abstract)

Location: `src/hydra/auth/credential_store.py`

Pluggable backend for credential retrieval. Default implementation reads from YAML. Future implementations may read from AWS Secrets Manager, HashiCorp Vault, or environment variables.

**Abstract methods:**

| Method | Signature | Returns | Description |
|---|---|---|---|
| `get` | `def get(self, stream_id: str) -> dict[str, Any]` | `dict` | Retrieve credential dict for a stream. Raises `CredentialNotFoundError` if absent. |
| `reload` | `def reload(self) -> None` | `None` | Re-read the backing store. Called on credential rotation events. |
| `list_streams` | `def list_streams(self) -> list[str]` | `list[str]` | Return all stream_ids with stored credentials. |

### 4.4 Class: `YamlCredentialStore(CredentialStore)`

Location: `src/hydra/auth/credential_store.py`

**Constructor parameters:**
- `path: Path` — path to the credentials YAML file. Defaults to `HydraSettings.credential_store_path`.

**Behavior:**
- On init, loads the YAML file into an in-memory dict keyed by `stream_id`.
- `get(stream_id)` returns the credential dict or raises `CredentialNotFoundError`.
- `reload()` re-reads the YAML file from disk, replacing the in-memory dict. Thread-safe via `threading.Lock`.
- `list_streams()` returns sorted list of keys.
- File-not-found on init raises `CredentialStoreError` with the path in the message.
- Malformed YAML raises `CredentialStoreError` wrapping the parse exception.

**Credential file schema** (`config/credentials.yaml`):

  client_secret: "oauth_client_secret"
  token_url: "https://auth.eurostat.ec.europa.eu/oauth/token"

scrape_icg:
  cookie_name: "session_id"
  cookie_value: "abc123def456"

adsb_exchange:
  rapidapi_key: "key_value"

### 4.5 Class: `AuthStrategy` (abstract)

Location: `src/hydra/auth/strategies.py`

**Constructor parameters:**
- `stream_id: str`
- `stream_config: dict` — the stream's entry from `stream_registry.yaml` (contains `auth_pattern`, `auth_key_location`, `auth_key_name`, `auth_token_location`, `cert_path`, `key_path`, etc.).
- `credentials: dict[str, Any]` — the credential dict from `CredentialStore.get(stream_id)`.

**Abstract methods:**

| Method | Signature | Returns | Description |
|---|---|---|---|
| `apply` | `async def apply(self) -> AuthenticatedRequest` | `AuthenticatedRequest` | Produce the auth context for an outbound request. Stateful strategies may acquire/refresh tokens here. |
| `on_401` | `async def on_401(self) -> AuthenticatedRequest | None` | `AuthenticatedRequest | None` | Called when the adapter receives HTTP 401. Stateful strategies refresh and return new auth context. Stateless strategies return `None` (no retry). |

**Concrete property:**

| Property | Type | Description |
|---|---|---|
| `is_stateful` | `bool` | `True` if the strategy manages token lifecycle. Used by `AuthManager` to determine caching behavior. |

### 4.6 Concrete Strategies

#### 4.6.1 `NoAuthStrategy`

Pattern: `none`. Returns empty `AuthenticatedRequest`. `on_401` returns `None`. `is_stateful = False`.

#### 4.6.2 `ApiKeyStrategy`

Pattern: `api_key`. `is_stateful = False`.

**`apply()`:**
1. Read `api_key` from credentials.
2. Read `auth_key_location` and `auth_key_name` from stream config.
3. If `auth_key_location == "header"`: set `headers[auth_key_name] = api_key`.
4. If `auth_key_location == "query"`: set `query_params[auth_key_name] = api_key`.
5. Return `AuthenticatedRequest`.

**`on_401()`:** Return `None` — API keys don't refresh.

**Validation on init:**
- `auth_key_location` must be present and one of `["header", "query"]`. Raise `AuthConfigError` otherwise.
- `auth_key_name` must be present and non-empty. Raise `AuthConfigError` otherwise.
- `api_key` must be present in credentials. Raise `CredentialNotFoundError` otherwise.

#### 4.6.3 `BasicAuthStrategy`

Pattern: `basic_auth`. `is_stateful = False`.

**`apply()`:**
1. Read `username` and `[PASSWORD]word` from credentials.
2. Encode as Base64: `base64(f"{username}:{[PASSWORD]word}")`.
3. Set `headers["Authorization"] = f"Basic {encoded}"`.
4. Return `AuthenticatedRequest`.

**`on_401()`:** Return `None`.

**Validation on init:**
- `username` and `[PASSWORD]word` must be present in credentials. Raise `CredentialNotFoundError` otherwise.

#### 4.6.4 `OAuth2ClientCredentialsStrategy`

Pattern: `oauth2_client_credentials`. `is_stateful = True`.

**State:**
- `_access_token: str | None` — cached token.
- `_token_expiry: datetime | None` — UTC expiry time.
- `_lock: asyncio.Lock` — prevents concurrent token refresh.

**`apply()`:**
1. Acquire lock.
2. If `_access_token` is `None` or `_token_expiry` is within 60 seconds of now (buffer): call `_acquire_token()`.
3. Set `headers["Authorization"] = f"Bearer {_access_token}"`.
4. Return `AuthenticatedRequest`.

**`_acquire_token()`:**
1. Read `client_id`, `client_secret`, `token_url` from credentials.
2. POST to `token_url` with `grant_type=client_credentials`, `client_id`, `client_secret` as form data.
3. Parse response JSON: extract `access_token` and `expires_in`.
4. Set `_access_token = access_token`.
5. Set `_token_expiry = utcnow() + timedelta(seconds=expires_in) - timedelta(seconds=60)`.
6. On HTTP error: raise `TokenAcquisitionError` with status code and response body.
7. On missing `access_token` in response: raise `TokenAcquisitionError`.

**`on_401()`:**
1. Invalidate `_access_token` (set to `None`).
2. Call `_acquire_token()`.
3. Return new `AuthenticatedRequest` with refreshed token.
4. If `_acquire_token()` fails, raise `TokenRefreshError`.

**Validation on init:**
- `client_id`, `client_secret`, `token_url` must be present in credentials. Raise `CredentialNotFoundError` otherwise.
- `token_url` must be a valid HTTPS URL. Raise `AuthConfigError` otherwise.

#### 4.6.5 `CookieAuthStrategy`

Pattern: `cookie_auth`. `is_stateful = False`.

**`apply()`:**
1. Read `cookie_name` and `cookie_value` from credentials.
2. Set `cookies[cookie_name] = cookie_value`.
3. Return `AuthenticatedRequest`.

**`on_401()`:** Return `None`.

**Validation on init:**
- `cookie_name` and `cookie_value` must be present in credentials. Raise `CredentialNotFoundError` otherwise.

#### 4.6.6 `RapidApiKeyStrategy`

Pattern: `rapidapi_key`. `is_stateful = False`.

**`apply()`:**
1. Read `rapidapi_key` and `rapidapi_host` from credentials.
2. Set `headers["X-RapidAPI-Key"] = rapidapi_key`.
3. Set `headers["X-RapidAPI-Host"] = rapidapi_host`.
4. Return `AuthenticatedRequest`.

**`on_401()`:** Return `None`.

**Validation on init:**
- `rapidapi_key` and `rapidapi_host` must be present in credentials. Raise `CredentialNotFoundError` otherwise.

#### 4.6.7 `CertificateAuthStrategy`

Pattern: `certificate`. `is_stateful = False`.

**`apply()`:**
1. Read `cert_path` and `key_path` from stream config (not credentials — these are file paths declared in the registry).
2. Construct `ssl.SSLContext(ssl.PROTOCOL_TLS_CLIENT)`.
3. Load cert chain: `ssl_context.load_cert_chain(certfile=cert_path, keyfile=key_path)`.
4. Set `ssl_context` on `AuthenticatedRequest`.
5. Return `AuthenticatedRequest`.

**`on_401()`:** Return `None` — certificate issues are not recoverable via retry.

**Validation on init:**
- `cert_path` and `key_path` must be present in stream config. Raise `AuthConfigError` otherwise.
- Both paths must point to existing files. Raise `AuthConfigError` with specific path if not found.

**Note:** The SSL context is constructed once on first `apply()` call and cached for the strategy's lifetime. Certificate files are validated on init but the SSL context is lazy-loaded to avoid filesystem operations during construction.

#### 4.6.8 `AccountTokenStrategy`

Pattern: `account_token`. `is_stateful = True` (for Earthdata OAuth2 variant); `False` for static tokens.

**Subvariants determined by credential structure:**

**Static token variant** (credentials contain `account_token` key):
1. Read `account_token` from credentials.
2. Read `auth_token_location` from stream config.
3. If `auth_token_location == "header"`: set `headers["Authorization"] = f"Bearer {account_token}"`.
4. If `auth_token_location == "cookie"`: set `cookies["account_token"] = account_token`.
5. `is_stateful = False`. `on_401()` returns `None`.

**Earthdata OAuth2 variant** (credentials contain `client_id`, `client_secret`, `username`, `[PASSWORD]word`):
1. `is_stateful = True`.
2. On `apply()`: if no cached token or token expired, execute Earthdata OAuth2 flow:
   a. POST to `https://urs.earthdata.nasa.gov/oauth/authorize` with client credentials.
   b. Exchange authorization code for bearer token.
   c. Cache token with expiry.
3. Set `headers["Authorization"] = f"Bearer {token}"`.
4. `on_401()`: invalidate cached token, re-acquire, return new `AuthenticatedRequest`.

**Variant detection on init:**
- If credentials contain `account_token`: static variant.
- If credentials contain `client_id` + `username`: Earthdata variant.
- If neither: raise `CredentialNotFoundError`.

**Validation on init:**
- Static: `account_token` must be non-empty string. `auth_token_location` must be in `["header", "cookie"]`.
- Earthdata: `client_id`, `client_secret`, `username`, `[PASSWORD]word` must all be present.

#### 4.6.9 `AwsCredentialsStrategy`

Pattern: `aws_credentials`. `is_stateful = True` (STS session tokens may expire; boto3 handles refresh internally).

**`apply()`:**
1. Read `aws_access_key_id`, `aws_secret_access_key`, and optional `aws_session_token` from credentials.
2. Construct `boto3_config` dict:
   ```python
   {
       "aws_access_key_id": creds["aws_access_key_id"],
       "aws_secret_access_key": creds["aws_secret_access_key"],
       "aws_session_token": creds.get("aws_session_token"),
   }
   ```
3. Return `AuthenticatedRequest` with `boto3_config` populated.

**Public bucket variant** (credentials empty or `auth_pattern == "none"` for S3 streams):
- Handled by `NoAuthStrategy` — the S3/Bulk adapter detects `boto3_config is None` and uses `botocore.UNSIGNED`.

**`on_401()`:** Return `None` — AWS credential refresh is handled by boto3's credential chain, not by HYDRA.

**Validation on init:**
- `aws_access_key_id` and `aws_secret_access_key` must be present. Raise `CredentialNotFoundError` otherwise.

### 4.7 Class: `AuthManager`

Location: `src/hydra/auth/manager.py`

The single entry point for all adapter auth operations. One `AuthManager` instance per adapter instance.

**Constructor parameters:**
- `stream_id: str`
- `stream_config: dict` — the stream's registry entry.
- `credential_store: CredentialStore` — injected store instance.

**Initialization:**
1. Read `auth_pattern` from `stream_config`. Default to `"none"` if absent.
2. Resolve credentials: `credential_store.get(stream_id)`. For `auth_pattern == "none"`, credentials may be absent — use empty dict.
3. Instantiate the appropriate `AuthStrategy` subclass via `_STRATEGY_REGISTRY` dispatch map.
4. Store the strategy instance.

**Strategy dispatch map** (`_STRATEGY_REGISTRY`):

```python
_STRATEGY_REGISTRY: dict[str, type[AuthStrategy]] = {
    "none": NoAuthStrategy,
    "api_key": ApiKeyStrategy,
    "basic_auth": BasicAuthStrategy,
    "oauth2_client_credentials": OAuth2ClientCredentialsStrategy,
    "cookie_auth": CookieAuthStrategy,
    "rapidapi_key": RapidApiKeyStrategy,
    "certificate": CertificateAuthStrategy,
    "account_token": AccountTokenStrategy,
    "aws_credentials": AwsCredentialsStrategy,
}
```

**Methods:**

| Method | Signature | Returns | Description |
|---|---|---|---|
| `apply` | `async def apply(self) -> AuthenticatedRequest` | `AuthenticatedRequest` | Delegates to `strategy.apply()`. |
| `handle_401` | `async def handle_401(self) -> AuthenticatedRequest | None` | `AuthenticatedRequest | None` | Delegates to `strategy.on_401()`. Returns `None` if strategy is stateless (no retry). |
| `reload_credentials` | `def reload_credentials(self) -> None` | `None` | Calls `credential_store.reload()`, re-resolves credentials for this stream, re-initializes the strategy. Used for credential rotation without adapter restart. |

**Error handling:**
- Unknown `auth_pattern` value: raise `AuthConfigError` with the pattern name and stream_id.
- `CredentialNotFoundError` from store: propagate — adapter's `run()` catches and logs.

## 5. Exception Hierarchy

Location: `src/hydra/auth/exceptions.py`

Extends the adapter exception hierarchy from P1.

```
AdapterError (from P1)
└── AuthError (new — base for all auth failures)
    ├── CredentialNotFoundError — stream_id has no credentials in the store
    ├── CredentialStoreError — store backend failure (file not found, parse error, connection error)
    ├── AuthConfigError — stream registry auth fields invalid or missing
    ├── TokenAcquisitionError — OAuth2/Earthdata token endpoint returned error
    └── TokenRefreshError — token refresh failed after 401
```

All auth exceptions carry `stream_id` as an attribute for structured logging.

## 6. BaseAdapter Integration

### 6.1 Modified `BaseAdapter.__init__`

After P6, `BaseAdapter.__init__` gains:

```python
def __init__(self, stream_id: str, settings: HydraSettings):
    # ... existing init (registry load, type validation) ...
    
    # NEW: Initialize auth manager
    credential_store = YamlCredentialStore(path=settings.credential_store_path)
    self._auth_manager = AuthManager(
        stream_id=stream_id,
        stream_config=self._stream_config,
        credential_store=credential_store,
    )
```

### 6.2 Modified `BaseAdapter` Fetch Wrapper

The retry wrapper in `BaseAdapter` that wraps subclass `fetch()` calls gains 401 handling:

```python
async def _fetch_with_retry(self) -> RawPayload:
    for attempt in range(self._max_retries):
        try:
            auth_context = await self._auth_manager.apply()
            raw = await self.fetch(auth_context=auth_context)
            return raw
        except FetchError as e:
            if e.http_status == 401 and self._auth_manager.strategy.is_stateful:
                refreshed = await self._auth_manager.handle_401()
                if refreshed is not None:
                    # Retry once with refreshed auth
                    raw = await self.fetch(auth_context=refreshed)
                    return raw
                else:
                    raise
            elif e.http_status == 429:
                # existing rate limit handling
                ...
            elif 500 <= e.http_status < 600:
                # existing retry with backoff
                ...
            else:
                raise
```

### 6.3 Modified Subclass `fetch` Signature

After P6, all adapter `fetch` methods accept an `AuthenticatedRequest` parameter:

```python
async def fetch(self, auth_context: AuthenticatedRequest) -> RawPayload:
```

The adapter merges `auth_context.headers` into its request headers, `auth_context.query_params` into its query parameters, `auth_context.cookies` into its cookie jar, and [PASSWORD]es `auth_context.ssl_context` to the HTTP client. For S3/Bulk adapter, `auth_context.boto3_config` is [PASSWORD]ed to `boto3.client()` construction.

This is a **breaking change** to the `fetch` abstract method signature. All adapters from P1–P5 must update their `fetch` signatures to accept `auth_context`. The inline auth dispatch logic in each adapter's `fetch` method is removed — replaced by the `auth_context` merge.

### 6.4 Adapter Migration Checklist

| Adapter | Remove | Add |
|---|---|---|
| RestJsonAdapter (P1) | Inline `api_key` header/query injection | Merge `auth_context.headers` and `auth_context.query_params` |
| CkanAdapter (P2) | Inline `api_key` injection | Merge `auth_context.headers` and `auth_context.query_params` |
| FdsnAdapter (P2) | No auth code to remove | Accept `auth_context` (will always be empty for `none`) |
| TapVoAdapter (P3) | Inline `account_token` cookie/header injection | Merge `auth_context.headers` and `auth_context.cookies` |
| S3BulkAdapter (P3) | Inline AWS credential and Earthdata OAuth2 logic | Use `auth_context.boto3_config` for boto3 client; merge `auth_context.headers` for HTTP downloads |
| OdataAdapter (P4) | Inline OAuth2 token acquisition, caching, 401 refresh | Merge `auth_context.headers` (token managed by strategy) |
| SdmxAdapter (P4) | Inline OAuth2 token acquisition, caching | Merge `auth_context.headers` |
| ScrapeRssAdapter (P5) | Inline `api_key`, `basic_auth`, `cookie_auth` dispatch | Merge `auth_context.headers` and `auth_context.cookies` |
| AisAdsbAdapter (P5) | Inline `api_key`, `basic_auth`, `rapidapi_key` dispatch | Merge `auth_context.headers` |
| StixTaxiiAdapter (P5) | Inline `api_key`, `basic_auth`, `certificate` dispatch | Merge `auth_context.headers` and `auth_context.ssl_context` |

## 7. Credential Store Extensibility

### 7.1 Future Store Implementations (Not in P6 MVP)

The `CredentialStore` abstract class is designed for future backends:

| Store | Backend | Use Case |
|---|---|---|
| `YamlCredentialStore` | Local YAML file | Development, single-node deployment (P6 MVP) |
| `EnvCredentialStore` | Environment variables | Container deployments, CI/CD (future) |
| `SecretsManagerStore` | AWS Secrets Manager | Production AWS deployments (future) |
| `VaultCredentialStore` | HashiCorp Vault | Multi-cloud production (future) |

### 7.2 Store Selection

`HydraSettings` gains a new field:

| Field | Type | Default | Description |
|---|---|---|---|
| `credential_store_type` | `enum[yaml, env, secrets_manager, vault]` | `yaml` | Which credential store backend to use. |

P6 MVP implements only `yaml`. The `AuthManager` factory reads this field and instantiates the appropriate store. Unknown store types raise `CredentialStoreError`.

## 13. Security Considerations

1. **Credential file permissions** — `YamlCredentialStore` logs a WARNING if the credential file has world-readable permissions (mode `o+r` on Unix). Does not enforce — operator responsibility.
2. **No credential logging** — Auth strategies NEVER log credential values (API keys, tokens, passwords, secrets). Only stream_id, pattern name, and non-sensitive metadata are logged.
3. **Token memory only** — OAuth2 tokens are cached in adapter instance memory. They are not persisted to disk, not written to logs, and not included in `NormalizedRecord` provenance.
4. **SSL verification** — `auth_ssl_verify` defaults to `True`. Setting to `False` logs a WARNING on every `apply()` call for that stream.
5. **Certificate key protection** — `CertificateAuthStrategy` does not read or cache the private key contents. It passes file paths to `ssl.SSLContext.load_cert_chain()`, which handles key material internally.
