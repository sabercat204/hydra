## 1. Purpose

Deliver two specialized adapters:
- **CKAN Adapter** — covers ~15 national open-data portals plus HDX (Humanitarian Data Exchange). Tiers 7, 9–13, 28.
- **FDSN Adapter** — covers seismic network federation services (IRIS, GEOFON, CTBTO). Tiers 1, 24.

Both subclass `BaseAdapter` from P1 and produce `NormalizedRecord` output.

## 2. CKAN Adapter — `src/hydra/adapters/ckan.py`

### 2.1 Class: `CkanAdapter(BaseAdapter)`

CKAN portals expose a standardized Action API (`/api/3/action/`). This adapter targets `package_search`, `package_show`, and `datastore_search` endpoints.

### 2.2 `fetch` Implementation

- Reads `base_url` (the CKAN portal root, e.g., `https://data.gov.au`) and `ckan_action` from the stream registry entry.
- Three fetch modes based on `ckan_action` field:
  - **`package_search`** — searches for datasets matching registry-declared `search_query` and `filter_tags`. Returns dataset metadata. Paginated via `start` + `rows` parameters.
  - **`package_show`** — fetches a specific dataset by `package_id` declared in registry. Returns full dataset metadata including resource URLs.
  - **`datastore_search`** — queries a specific CKAN DataStore resource by `resource_id`. Returns tabular data. Paginated via `offset` + `limit`.
- After metadata fetch, if the registry entry declares `download_resources: true`, the adapter follows resource URLs to download the actual data files (CSV, JSON, XLS). Downloads are stored as additional `RawPayload` entries.
- Auth: most CKAN portals are open (`auth_pattern: none`). Some require account tokens (`auth_pattern: api_key`), handled identically to REST/JSON adapter.
- Pagination: follows CKAN's `count` / `start` / `rows` pattern until all results retrieved or `max_pages` reached.

### 2.3 `parse` Implementation

- For `package_search` / `package_show` responses: extracts dataset metadata (title, description, resources, tags, organization, update frequency) into dicts.
- For `datastore_search` responses: extracts the `records` array directly — each record is already a flat dict.
- For downloaded resource files:
  - CSV: parsed via Python `csv` module into list of dicts.
  - JSON: parsed directly.
  - XLS/XLSX: parsed via `openpyxl` into list of dicts (first row as headers).
- Each parsed record is tagged with `portal_id` (the CKAN portal identifier from registry) and `resource_id`.

### 2.4 `validate` Implementation

- Required-field validation per registry `required_fields`.
- Schema consistency check: if the first record establishes a field set, subsequent records missing >50% of those fields are flagged as schema-drift and logged at WARNING.
- Encoding normalization: force UTF-8, handle common CKAN portal encoding issues (Latin-1 mislabeled as UTF-8).

### 2.5 Stream Registry Entry Schema (CKAN-specific fields)

| Field | Type | Required | Description |
|---|---|---|---|
| `ckan_action` | `enum[package_search, package_show, datastore_search]` | yes | Which CKAN API action to invoke. |
| `search_query` | `str` | no | Free-text search query for `package_search`. |
| `filter_tags` | `list[str]` | no | Tag filters for `package_search`. |
| `package_id` | `str` | no | Specific dataset ID for `package_show`. |
| `resource_id` | `str` | no | Specific resource ID for `datastore_search`. |
| `download_resources` | `bool` | no | Whether to follow resource URLs and download files. Default: false. |
| `resource_format_filter` | `list[str]` | no | Only download resources matching these formats (e.g., `["csv", "json"]`). |

## 3. FDSN Adapter — `src/hydra/adapters/fdsn.py`

### 3.1 Class: `FdsnAdapter(BaseAdapter)`

The International Federation of Digital Seismograph Networks (FDSN) defines a standardized web service specification. All compliant data centers (IRIS, GEOFON, CTBTO, etc.) expose identical endpoints with identical query parameters.

### 3.2 FDSN Service Endpoints

Three services, selected via registry `fdsn_service` field:

| Service | Path | Returns | Use Case |
|---|---|---|---|
| `fdsnws-event` | `/fdsnws/event/1/query` | QuakeML (XML) | Earthquake event catalog queries. |
| `fdsnws-station` | `/fdsnws/station/1/query` | StationXML | Station metadata and instrument response. |
| `fdsnws-dataselect` | `/fdsnws/dataselect/1/query` | miniSEED (binary) | Raw waveform time-series data. |

### 3.3 `fetch` Implementation

- Reads `base_url` (e.g., `https://service.iris.edu`) and `fdsn_service` from registry.
- Constructs query parameters from registry-declared values and dynamic parameters:
  - Common: `starttime`, `endtime` (ISO 8601), `minmagnitude`, `maxmagnitude`, `minlatitude`, `maxlatitude`, `minlongitude`, `maxlongitude`.
  - Dynamic: `starttime` defaults to `{last_fetch_time}` if not explicitly set — the scheduler provides this.
  - `format` parameter: set to `xml` for event/station, `miniseed` for dataselect.
- Response content types:
  - `fdsnws-event`: `application/xml` (QuakeML).
  - `fdsnws-station`: `application/xml` (StationXML).
  - `fdsnws-dataselect`: `application/vnd.fdsn.mseed` (binary miniSEED).
- HTTP 204 (No Content) is a valid FDSN response meaning "no data matches your query" — handle gracefully, return empty `RawPayload`.
- HTTP 413 (Request Too Large) — FDSN services return this when the query would produce too much data. The adapter must split the time window in half and retry both halves (recursive bisection, max depth from registry `max_bisect_depth`, default 3).

### 3.4 `parse` Implementation

- **QuakeML (event service):** Parse XML using `xml.etree.ElementTree`. Extract event elements: `publicID`, `origin` (time, latitude, longitude, depth), `magnitude` (mag, type), `description`, `creationInfo`. Each event becomes one dict.
- **StationXML (station service):** Parse XML. Extract network → station → channel hierarchy. Each channel becomes one dict with fields: `network_code`, `station_code`, `channel_code`, `location_code`, `latitude`, `longitude`, `elevation`, `start_date`, `end_date`, `sample_rate`.
- **miniSEED (dataselect service):** This is a binary format. The adapter does NOT parse miniSEED into individual samples at ingestion time — that would be prohibitively expensive. Instead:
  - Extract trace-level metadata from miniSEED headers: `network`, `station`, `channel`, `location`, `starttime`, `endtime`, `sample_rate`, `num_samples`.
  - Store the raw binary in the `RawPayload` for downstream storage in S3-compatible object store.
  - Each trace segment becomes one dict with metadata fields only.
  - Note in the registry: `parser: miniseed_header_only` to signal this behavior.

### 3.5 `validate` Implementation

- Event records: validate that `latitude` is in [-90, 90], `longitude` in [-180, 180], `depth` is non-negative, `magnitude` is numeric.
- Station records: validate coordinate ranges, `sample_rate` > 0.
- miniSEED metadata: validate `num_samples` > 0, `starttime` < `endtime`.
- Deduplication: use `publicID` for events, `network.station.channel.location.starttime` composite key for waveforms.

### 3.6 Stream Registry Entry Schema (FDSN-specific fields)

| Field | Type | Required | Description |
|---|---|---|---|
| `fdsn_service` | `enum[event, station, dataselect]` | yes | Which FDSN service to query. |
| `fdsn_query_params` | `dict` | no | Static query parameters (e.g., `minmagnitude: 4.0`). |
| `max_bisect_depth` | `int` | no | Max recursive time-window splits on HTTP 413. Default: 3. |
| `parser` | `str` | no | Parser hint. `miniseed_header_only` for dataselect streams. |

## 4. Specialized Parser Dependencies

| Parser | Library | Adapter | Notes |
|---|---|---|---|
| QuakeML | `xml.etree.ElementTree` (stdlib) | FDSN | No external dependency. |
| StationXML | `xml.etree.ElementTree` (stdlib) | FDSN | No external dependency. |
| miniSEED headers | `obspy` | FDSN | Required for binary header extraction. Add to `pyproject.toml` as optional dependency group `[seismic]`. |

**New dependency:** `obspy` — added to `pyproject.toml` under `[project.optional-dependencies]` group `seismic`. Not required for non-seismic adapters.

## 5. Test Specification

### 5.1 `tests/test_ckan_adapter.py`

- Test `package_search` fetch + parse with mocked CKAN API response (Australian data.gov.au format).
- Test `datastore_search` with pagination — mock 3 pages of tabular data, verify all accumulated.
- Test resource download: mock `package_show` returning a dataset with CSV resource URL, verify CSV downloaded and parsed.
- Test encoding normalization: inject Latin-1 encoded response with UTF-8 content-type header, verify graceful handling.
- Test schema-drift detection: inject records where later records are missing >50% of fields from first record, verify WARNING logged.
- Test auth: mock a portal requiring API key, verify key injected.

### 5.2 `tests/test_fdsn_adapter.py`

- Test event service: mock IRIS QuakeML response with 3 earthquakes, verify 3 dicts extracted with correct fields.
- Test station service: mock StationXML with network → station → channel hierarchy, verify flat channel dicts.
- Test dataselect service: mock miniSEED binary (use a small fixture file), verify header-only metadata extraction.
- Test HTTP 204 handling: verify empty list returned, no error raised.
- Test HTTP 413 bisection: mock 413 on full time range, 200 on each half, verify both halves fetched and results merged.
- Test coordinate validation: inject event with latitude 95.0, verify it is dropped.
- Test deduplication: inject two events with same `publicID`, verify only one passes.
