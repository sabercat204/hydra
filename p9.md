## 1. Purpose

Build the cross-tier correlation layer that discovers relationships between records ingested by independent adapters. P9 owns:

1. **Pipeline framework** — pluggable correlation pipelines that consume `NormalizedRecord` batches from PostgreSQL and produce `CorrelationResult` objects.
2. **Three MVP pipelines** — geospatial-temporal, entity-network, and threat-convergence — matching the three `correlation_pipelines` declared in P0's `stream_registry.yaml`.
3. **Matching strategies** — reusable matchers for spatial proximity, temporal co-occurrence, entity identity resolution, and tag/keyword overlap.
4. **Confidence scoring** — composite scoring model that weights match quality across dimensions.
5. **Trigger integration** — post-ingestion hooks from P8 cadence DAGs that invoke correlation on fresh data.
6. **Result persistence** — correlation outputs stored in PostgreSQL (relational) and Neo4j (graph edges).

After P9, the platform produces intelligence — not just data. Cross-tier relationships surface patterns invisible within any single tier.

## 3. Architecture

### 3.1 Data Flow

```
Trigger (P8 post-ingestion or scheduled DAG)
       │
       ▼
CorrelationEngine.run(pipeline_id, params)
       │
       ├── 1. Query PostgreSQL for candidate records (time window, tiers)
       ├── 2. Optional: Query Elasticsearch for text-based candidates
       ├── 3. Pipeline.correlate(candidates) → list[CorrelationResult]
       │       ├── Matcher.match(record_a, record_b) → MatchScore
       │       ├── Scorer.composite(match_scores) → confidence
       │       └── Filter by confidence threshold
       ├── 4. Deduplicate results (hash of sorted record pair + pipeline_id)
       ├── 5. Persist to PostgreSQL (correlation_results table)
       └── 6. Persist to Neo4j (cross-tier CORRELATED_WITH edges)
```

## 5. Correlation Engine

Location: `src/hydra/correlation/engine.py`

### 5.1 Class Definition

```python
class CorrelationEngine:
    """
    Orchestrates correlation pipeline execution.

    Responsibilities:
    1. Query candidate records from PostgreSQL (and optionally Elasticsearch).
    2. Dispatch to the appropriate pipeline.
    3. Deduplicate results.
    4. Persist to PostgreSQL and Neo4j.
    """

    def __init__(
        self,
        pg_engine: PostgresEngine,
        neo4j_engine: Neo4jEngine,
        es_engine: ElasticsearchEngine | None,
        registry: StreamRegistry,
        settings: HydraSettings,
    ):
        self._pg = pg_engine
        self._neo4j = neo4j_engine
        self._es = es_engine
        self._registry = registry
        self._settings = settings
        self._pipelines: dict[str, BasePipeline] = {}

    def register_pipeline(self, pipeline: BasePipeline) -> None:
        """Register a correlation pipeline by its ID."""
        self._pipelines[pipeline.pipeline_id] = pipeline

    async def run(
        self,
        pipeline_id: str,
        time_window_start: str | None = None,
        time_window_end: str | None = None,
        trigger_tiers: list[int] | None = None,
    ) -> CorrelationRunResult:
        """
        Execute a correlation pipeline.

        Steps:
        1. Resolve pipeline from registry.
        2. Build candidate set — query PG for records in the pipeline's
           source tiers within the time window.
        3. Call pipeline.correlate(candidates).
        4. Deduplicate results against existing correlations.
        5. Persist new results to PG and Neo4j.
        6. Return run summary.

        Parameters:
        - pipeline_id: which pipeline to run
        - time_window_start/end: override default lookback window
        - trigger_tiers: if triggered by P8, which tiers have fresh data
          (used to optimize candidate queries — only cross-correlate
          fresh records against existing, not all-vs-all)
        """
        ...

    async def _build_candidate_set(
        self,
        pipeline_id: str,
        source_tiers: list[int],
        time_window_start: str,
        time_window_end: str,
        trigger_tiers: list[int] | None,
    ) -> CandidateSet:
        """
        Query PostgreSQL for candidate records.

        If trigger_tiers is set: query fresh records from trigger tiers
        (ingested since last correlation run) and existing records from
        other source tiers within the time window. This avoids O(n²)
        all-vs-all on every run.

        If trigger_tiers is None (scheduled run): query all source tiers
        within the time window.
        """
        ...

    async def _persist_results(
        self, results: list[CorrelationResult]
    ) -> PersistResult:
        """
        Write correlation results to PostgreSQL and Neo4j.

        PG: INSERT into correlation_results table.
        Neo4j: MERGE cross-tier CORRELATED_WITH edges between
               existing nodes (if both records have Neo4j nodes)
               or create lightweight reference nodes for records
               not in graph tiers.
        """
        ...

    async def _deduplicate(
        self, results: list[CorrelationResult]
    ) -> list[CorrelationResult]:
        """
        Remove results that already exist in PG (by correlation_hash).

        Also handles confidence updates: if a new result has higher
        confidence than an existing correlation for the same pair,
        update rather than skip.
        """
        ...
```

## 7. MVP Pipelines

### 7.1 Geospatial-Temporal Pipeline

Location: `src/hydra/correlation/pipelines/geospatial_temporal.py`

**Purpose:** Discover records from different tiers that co-occur in space and time. A seismic event (Tier 1) near a conflict event (Tier 15) within a time window may indicate underground nuclear testing. Maritime traffic (Tier 18) near a sanctions-listed port (Tier 19) within hours of a sanctions update is operationally significant.

**Source tiers:** 1 (Geophysical), 2 (Atmospheric), 3 (Space Weather), 15 (Conflict), 18 (Aviation & Maritime), 20 (NBC Threat), 23 (Space Situational Awareness), 24 (Geoscience), 25 (Environmental)

```python
class GeospatialTemporalPipeline(BasePipeline):
    pipeline_id = "geospatial_temporal"
    source_tiers = [1, 2, 3, 15, 18, 20, 23, 24, 25]

    def __init__(self, settings: HydraSettings):
        self._spatial_radius_km = settings.correlation.geo_temporal_radius_km  # default: 50
        self._temporal_window_s = settings.correlation.geo_temporal_window_s   # default: 3600 (1 hour)
        self._matchers = [
            SpatialProximityMatcher(max_distance_km=self._spatial_radius_km),
            TemporalCooccurrenceMatcher(max_delta_s=self._temporal_window_s),
        ]
        self._scorer = CompositeScorer(weights={"spatial": 0.6, "temporal": 0.4})

    async def correlate(self, candidates: CandidateSet) -> list[CorrelationResult]:
        """
        Algorithm:
        1. Filter to records with non-null geo.
        2. Build spatial index (R-tree or brute-force for MVP).
        3. For each cross-tier pair within spatial radius:
           a. Check temporal co-occurrence.
           b. Score composite confidence.
           c. Emit if above threshold.
        """
        ...
```

**Spatial matching strategy:**

For MVP, use a two-phase approach:
1. **Coarse filter:** Group records by geohash prefix (precision 4 ≈ 39km × 20km cells). Records sharing a geohash prefix or adjacent prefixes are candidate pairs.
2. **Fine filter:** Haversine distance calculation on candidate pairs. Records within `spatial_radius_km` pass.

Post-MVP: R-tree spatial index via `rtree` library for O(n log n) performance.

**Cross-tier pair rules:**

Not all tier combinations are meaningful. The pipeline defines a tier affinity matrix:

| Tier A | Tier B | Rationale |
|---|---|---|
| 1 (Geophysical) | 20 (NBC Threat) | Seismic event near nuclear facility → possible underground test |
| 1 (Geophysical) | 15 (Conflict) | Seismic event in conflict zone → possible explosion |
| 18 (Aviation/Maritime) | 19 (Sanctions) | Vessel near sanctioned port/entity |
| 18 (Aviation/Maritime) | 15 (Conflict) | Aircraft/vessel in conflict zone |
| 2 (Atmospheric) | 20 (NBC Threat) | Atmospheric anomaly near NBC facility |
| 3 (Space Weather) | 23 (Space Situational) | Solar event coinciding with satellite anomaly |
| 24 (Geoscience) | 25 (Environmental) | Earth system event with environmental impact |
| 1 (Geophysical) | 24 (Geoscience) | Cross-source seismic/geoscience correlation |

Pairs not in the affinity matrix are skipped — prevents combinatorial explosion and meaningless correlations.

### 7.2 Entity Network Pipeline

Location: `src/hydra/correlation/pipelines/entity_network.py`

**Purpose:** Resolve entity identities across tiers. A sanctioned entity (Tier 19) appearing in conflict data (Tier 15), cyber threat intel (Tier 16), or human rights reports (Tier 21) represents the same real-world actor. Entity resolution links these records.

**Source tiers:** 15 (Conflict), 16 (Cyber Threat), 19 (Sanctions), 21 (Human Rights), 8 (International Orgs), 14 (Arms & Defense)

```python
class EntityNetworkPipeline(BasePipeline):
    pipeline_id = "entity_network"
    source_tiers = [8, 14, 15, 16, 19, 21]

    def __init__(
        self,
        settings: HydraSettings,
        es_engine: ElasticsearchEngine | None,
    ):
        self._es = es_engine
        self._matchers = [
            EntityNameMatcher(similarity_threshold=0.85),
            EntityIdMatcher(),       # exact ID match (STIX IDs, OFAC IDs, etc.)
            TagOverlapMatcher(min_overlap=2),
            KeywordCooccurrenceMatcher(min_shared_keywords=3),
        ]
        self._scorer = CompositeScorer(weights={
            "entity": 0.5,
            "keyword": 0.2,
            "tag": 0.15,
            "temporal": 0.15,
        })

    async def correlate(self, candidates: CandidateSet) -> list[CorrelationResult]:
        """
        Algorithm:
        1. Extract entity identifiers from each record's payload:
           - Tier 16: STIX object IDs, MITRE ATT&CK IDs, CVE IDs
           - Tier 19: OFAC entity IDs, OpenSanctions IDs, GLEIF LEIs
           - Tier 15: ACLED actor IDs, GDELT actor codes
           - Tier 21: OHCHR case IDs, ICC case numbers
           - Tier 14: SIPRI program IDs
           - Tier 8: UN resolution numbers, WHO event IDs
        2. Phase 1 — Exact ID match: cross-tier records sharing
           an identifier are high-confidence matches.
        3. Phase 2 — Fuzzy name match: extract entity names from
           payload, use Levenshtein/Jaro-Winkler similarity.
           Optional: Elasticsearch more_like_this query for
           text-based candidate discovery.
        4. Phase 3 — Tag and keyword overlap: records sharing
           tags or payload keywords above threshold.
        5. Score and emit.
        """
        ...
```

### 7.3 Threat Convergence Pipeline

Location: `src/hydra/correlation/pipelines/threat_convergence.py`

**Purpose:** Detect multi-signal threat patterns where independent indicators from different domains converge on a single threat assessment. A cyber attack (Tier 16) targeting energy infrastructure (Tier 27) in a country under sanctions (Tier 19) during a conflict escalation (Tier 15) represents a convergent threat pattern.

**Source tiers:** 15 (Conflict), 16 (Cyber Threat), 19 (Sanctions), 20 (NBC Threat), 6 (Law Enforcement), 27 (Energy)

```python
class ThreatConvergencePipeline(BasePipeline):
    pipeline_id = "threat_convergence"
    source_tiers = [6, 15, 16, 19, 20, 27]
    confidence_threshold = 0.6  # higher threshold — threat assessments need stronger signal

    def __init__(self, settings: HydraSettings):
        self._temporal_window_s = settings.correlation.threat_convergence_window_s  # default: 86400 (24h)
        self._matchers = [
            TemporalCooccurrenceMatcher(max_delta_s=self._temporal_window_s),
            EntityNameMatcher(similarity_threshold=0.80),
            TagOverlapMatcher(min_overlap=2),
            GeographicRegionMatcher(),  # country/region-level, not point proximity
        ]
        self._scorer = CompositeScorer(weights={
            "temporal": 0.25,
            "entity": 0.30,
            "tag": 0.15,
            "geographic_region": 0.30,
        })

    async def correlate(self, candidates: CandidateSet) -> list[CorrelationResult]:
        """
        Algorithm:
        1. Group records by geographic region (country code extracted
           from payload or reverse-geocoded from geo coordinates).
        2. Within each region, apply temporal windowing.
        3. For each cross-tier pair within the region+time window:
           a. Entity name matching (threat actors, target organizations).
           b. Tag overlap (shared threat categories, CVE references).
           c. Geographic region confirmation.
           d. Temporal proximity scoring.
        4. Multi-signal bonus: if a region+time window contains records
           from 3+ different source tiers, apply a convergence multiplier
           (1.2x) to all pairwise correlations in that cluster.
        5. Score and emit.
        """
        ...
```

**Convergence multiplier rationale:**

Two-tier correlations are common and may be coincidental. Three or more independent tiers reporting related activity in the same region and time window is a significantly stronger signal. The 1.2x multiplier (capped at 1.0 final confidence) rewards multi-source convergence without overriding weak individual matches.

## 9. Scoring

Location: `src/hydra/correlation/scoring.py`

### 9.1 CompositeScorer

```python
class CompositeScorer:
    """
    Weighted composite scoring across match dimensions.

    Each pipeline declares dimension weights that sum to 1.0.
    The composite score is the weighted sum of individual dimension scores.
    Missing dimensions (matcher returned None) contribute 0.0.
    """

    def __init__(self, weights: dict[str, float]):
        total = sum(weights.values())
        if abs(total - 1.0) > 0.01:
            raise ValueError(f"Weights must sum to 1.0, got {total}")
        self._weights = weights

    def score(self, match_scores: list[MatchScore]) -> float:
        """
        Compute weighted composite confidence.

        Returns value in [0.0, 1.0].
        """
        score_map = {ms.dimension: ms.score for ms in match_scores}
        composite = sum(
            self._weights.get(dim, 0.0) * score_map.get(dim, 0.0)
            for dim in self._weights
        )
        return min(max(composite, 0.0), 1.0)

    def score_with_convergence(
        self,
        match_scores: list[MatchScore],
        convergence_multiplier: float = 1.0,
    ) -> float:
        """
        Composite score with optional convergence bonus.

        Used by ThreatConvergencePipeline when 3+ tiers converge.
        Final score capped at 1.0.
        """
        base = self.score(match_scores)
        return min(base * convergence_multiplier, 1.0)
```

### 10.1 Post-Ingestion Triggers

P8 cadence DAGs fire correlation triggers after successful ingestion runs. The trigger mechanism:

```python
class CorrelationTrigger:
    """
    Handles post-ingestion correlation triggers from P8 cadence DAGs.

    When a cadence DAG completes ingestion for tiers that are source tiers
    for a correlation pipeline, the trigger evaluates whether to invoke
    the pipeline.
    """

    def __init__(
        self,
        engine: CorrelationEngine,
        registry: StreamRegistry,
        settings: HydraSettings,
    ):
        self._engine = engine
        self._registry = registry
        self._settings = settings
        self._pipeline_tier_map: dict[str, list[int]] = {
            "geospatial_temporal": [1, 2, 3, 15, 18, 20, 23, 24, 25],
            "entity_network": [8, 14, 15, 16, 19, 21],
            "threat_convergence": [6, 15, 16, 19, 20, 27],
        }

    async def on_ingestion_complete(
        self,
        completed_tiers: list[int],
        ingestion_timestamp: str,
    ) -> list[CorrelationRunResult]:
        """
        Called by P8 cadence DAG after successful ingestion.

        Determines which correlation pipelines should run based on
        which tiers just ingested fresh data.

        Logic:
        1. For each pipeline, check if any completed tier is in
           the pipeline's source tiers.
        2. If yes, trigger the pipeline with:
           - trigger_tiers = intersection of completed_tiers and source_tiers
           - time_window = lookback from ingestion_timestamp
             (default: 2x the cadence interval of the triggering tier)
        3. Collect and return all run results.

        Deduplication: if the same pipeline was triggered within
        the last `min_trigger_interval_s` (default: 300s), skip.
        This prevents sub_minute cadences from triggering correlation
        every 60 seconds.
        """
        ...

    def _should_trigger(self, pipeline_id: str) -> bool:
        """
        Check minimum trigger interval.

        Uses Redis key: hydra:correlation:last_trigger:{pipeline_id}
        If elapsed time < min_trigger_interval_s, return False.
        """
        ...

    def _compute_lookback_window(
        self, completed_tiers: list[int], ingestion_timestamp: str
    ) -> tuple[str, str]:
        """
        Compute time window for correlation query.

        Default: 2x the longest cadence interval among completed tiers.
        Capped at max_lookback_s (default: 86400 = 24 hours).

        Returns (start, end) as ISO 8601 UTC strings.
        """
        ...
```

### 10.2 Airflow Integration

Each cadence DAG gets a terminal task that fires the correlation trigger:

```python
def trigger_correlation_task(completed_tiers: list[int], **context):
    """
    Airflow PythonOperator callable — terminal task in each cadence DAG.

    Runs after all tier TaskGroups complete (success or partial).
    Collects completed tier IDs from XCom, fires CorrelationTrigger.
    """
    trigger = _get_correlation_trigger()  # singleton per worker
    execution_date = context["execution_date"].isoformat()
    results = asyncio.run(
        trigger.on_ingestion_complete(completed_tiers, execution_date)
    )

    for result in results:
        context["ti"].xcom_push(
            key=f"correlation_{result.pipeline_id}",
            value=asdict(result),
        )
```

Task dependency: all tier TaskGroups → `trigger_correlation_task` (trigger_rule: `none_failed_min_one_success`). This ensures correlation runs even if some tiers failed, as long as at least one tier succeeded.

### 10.3 Scheduled Correlation DAGs

In addition to triggered runs, each pipeline has a standalone scheduled DAG for full-window correlation (not just fresh data):

```
DAG: correlation_geospatial_temporal (schedule: @hourly)
└── task: run_geospatial_temporal_full

DAG: correlation_entity_network (schedule: @daily)
└── task: run_entity_network_full

DAG: correlation_threat_convergence (schedule: every 6 hours)
└── task: run_threat_convergence_full
```

Scheduled runs use the full lookback window (configurable per pipeline) and do not set `trigger_tiers` — they evaluate all source tier records within the window. This catches correlations missed by triggered runs (e.g., when both correlated records were ingested in different cadence cycles).

CREATE TABLE correlation_results (

CREATE INDEX idx_correlation_pipeline ON correlation_results(pipeline_id);

CREATE INDEX idx_correlation_confidence ON correlation_results(confidence DESC);
CREATE INDEX idx_correlation_record_a ON correlation_results(record_a_hash);
CREATE INDEX idx_correlation_record_b ON correlation_results(record_b_hash);
CREATE INDEX idx_correlation_created ON correlation_results(created_at);

### 11.2 Neo4j Graph Schema

New relationship type: `CORRELATED_WITH`

```cypher
// Cross-tier correlation edge
(:Record {raw_hash: $record_a_hash, tier: $tier_a})
  -[:CORRELATED_WITH {
      correlation_id: $correlation_id,
      pipeline_id: $pipeline_id,
      confidence: $confidence,
      match_dimensions: $match_dimensions,
      created_at: $created_at
  }]->
(:Record {raw_hash: $record_b_hash, tier: $tier_b})
```

For records not already in Neo4j (i.e., from non-graph tiers), the engine creates lightweight reference nodes:

```cypher
MERGE (r:Record {raw_hash: $raw_hash})
ON CREATE SET r.tier = $tier, r.stream_id = $stream_id,
             r.timestamp = $timestamp, r.created_at = $now
```

This ensures correlation edges can always be created regardless of whether the source tiers are graph-schema tiers (15, 16, 19) or not.

Edge directionality: `record_a → record_b` where `tier_a < tier_b` (canonical ordering prevents duplicate edges in opposite directions).

## 13. Exceptions

Location: `src/hydra/correlation/exceptions.py`

```python
class CorrelationError(Exception):
    """Base exception for correlation module."""

class PipelineNotFoundError(CorrelationError):
    """Raised when a requested pipeline_id is not registered."""
    def __init__(self, pipeline_id: str):
        self.pipeline_id = pipeline_id
        super().__init__(f"Unknown pipeline: {pipeline_id}")

class CandidateQueryError(CorrelationError):
    """Raised when candidate record query fails."""
    def __init__(self, pipeline_id: str, cause: str):
        self.pipeline_id = pipeline_id
        super().__init__(f"Candidate query failed for {pipeline_id}: {cause}")

class PersistenceError(CorrelationError):
    """Raised when correlation result persistence fails."""
    def __init__(self, target: str, cause: str):
        self.target = target
        super().__init__(f"Persistence failed ({target}): {cause}")

class MatcherError(CorrelationError):
    """Raised when a matcher encounters an unrecoverable error."""
    def __init__(self, dimension: str, cause: str):
        self.dimension = dimension
        super().__init__(f"Matcher error ({dimension}): {cause}")

class TriggerThrottledError(CorrelationError):
    """Raised when a correlation trigger is throttled by min_trigger_interval."""
    def __init__(self, pipeline_id: str, elapsed_s: float, min_interval_s: float):
        self.pipeline_id = pipeline_id
        super().__init__(
            f"Trigger throttled: {pipeline_id} last ran {elapsed_s:.0f}s ago "
            f"(min interval: {min_interval_s:.0f}s)"
        )
```
